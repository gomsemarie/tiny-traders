# Tiny Traders - 게임 기획서

## 1. 게임 개요

| 항목 | 내용 |
|------|------|
| **장르** | 도트 2D 시뮬레이션 (다마고치 + 모의투자 + 타이쿤) |
| **타겟** | 직장 동료들과 가볍게 즐길 수 있는 캐주얼 게임 |
| **개발 단계** | 1단계: 로컬 플레이 → 2단계: 멀티플레이 (데디케이티드 서버) |
| **서버 방식** | Docker 기반 데디케이티드 서버 (회사별 자체 호스팅) |
| **핵심 키워드** | 캐릭터 육성, 모의 주식투자, 아르바이트 미니게임, 거래 시스템 |
| **아트 스타일** | 도트(픽셀아트) 2D |

## 1.1 UI/UX 디자인 원칙 [확정]

이 게임의 성패를 좌우하는 **3대 UX 축**과 전체 비주얼 톤을 정의한다.

#### 1순위: 관리자 에디터 UX — "내가 편해야 게임이 산다"

관리자(서버 운영자)가 코드 한 줄 안 건드리고도 컨텐츠를 빠르게 추가/수정/배포할 수 있어야 한다. 관리자 에디터는 게임 본체만큼 중요한 제품이다.

| 원칙 | 설명 |
|------|------|
| **즉시 확인** | 수정 사항이 실시간 반영되어 별도 빌드/재시작 없이 결과를 바로 확인 |
| **일괄 편집** | Glide Data Grid 기반 스프레드시트형 편집 — 한 화면에서 여러 항목 동시 수정 |
| **인라인 검증** | 잘못된 값 입력 시 셀 단위로 즉시 오류 표시, 저장 전 경고 |
| **되돌리기** | 변경 이력 기반 Undo/Redo, 특정 시점으로 롤백 |
| **검색/필터** | 모든 관리 목록에서 즉시 검색, 다중 필터, 정렬 |
| **자동 패치노트** | 수정 내역이 자동으로 유저 게시판에 반영 — 별도 작성 불필요 |

#### 2순위: 모의투자 UI/UX — "실제 증권앱처럼, 하지만 게임답게"

모의투자가 게임의 핵심 컨텐츠이므로 투자 화면은 **실제 트레이딩 앱 수준의 정보량과 조작성**을 제공하되, 게임의 아기자기한 톤과 조화를 이뤄야 한다.

| 원칙 | 설명 |
|------|------|
| **전문적 차트** | 캔들차트, 거래량, 이동평균선 등 실전급 차트 제공 |
| **빠른 주문** | 시장가/지정가/손절/익절을 최소 클릭으로 실행 — 원터치 매매 |
| **포트폴리오 한눈에** | 보유 종목, 평가손익, 수익률, 자산 구성을 한 화면에 대시보드로 |
| **실시간 피드백** | 체결 알림, 손익 변동, 돌발 행동 알림이 즉각적으로 시각/청각 피드백 |
| **종목 탐색** | 상품 유형(주식/ETF/암호화폐/환율) 필터, 관심 종목, 최근 거래 — nuqs로 필터 상태 URL 유지 |
| **게임 요소 통합** | 차트 옆에 배치된 캐릭터 상태, 돌발 행동 경고, 스킬 버프 표시가 자연스럽게 녹아듦 |

#### 3순위: 소셜/멀티플레이 UX — "같이 해야 재미있다"

회사 동료끼리 같은 서버에서 즐기는 게임이므로, 다른 유저의 존재를 항상 느낄 수 있어야 한다.

| 원칙 | 설명 |
|------|------|
| **접속자 목록** | 항상 화면 한쪽에 접속 중인 동료 목록 (칭호, 순자산 포함) |
| **실시간 알림** | 대출 요청, 마켓 등록, 랭킹 변동, 채무불이행 등 주요 활동이 전체 피드에 표시 |
| **비교/경쟁** | 랭킹 보드, 포트폴리오 수익률 비교, 칭호 자랑 |
| **거래 편의** | 마켓/대출 게시판이 직관적이고, 등록-수락 흐름이 2~3클릭 이내로 완료 |
| **이벤트 공유** | 주기적 이벤트 발생 시 전원이 동시에 체감 — 공동 경험 |

#### 비주얼 톤: "귀엽고 아기자기한 도트"

| 항목 | 방향 |
|------|------|
| **캐릭터** | 둥글고 귀여운 2~3등신 도트 캐릭터, 다양한 표정/리액션 애니메이션 |
| **시설/배경** | 파스텔톤 타일맵, 미니어처 느낌의 건물, 작은 디테일(연기, 꽃, 동물 등) |
| **UI 톤** | shadcn/ui 기반이되 **라운드 모서리, 부드러운 색상, 도트 아이콘** 커스터마이징 |
| **이펙트** | 별, 반짝임, 하트, 코인 튀는 애니메이션 등 과하지 않은 귀여운 이펙트 |
| **투자 화면** | 차트/수치는 전문적이되, 프레임/배경은 게임 톤 유지 — "귀여운 껍데기 안에 진지한 내용" |
| **사운드** | 경쾌하고 가벼운 BGM, 도트 게임 특유의 효과음 (매수=찰칵, 수익=코인, 손실=삐빅) |

## 2. 핵심 시스템

### 2.1 캐릭터 육성 시스템 [확정]

#### 캐릭터의 3가지 용도

1. **수집과 육성의 재미** — 도트 캐릭터를 키우고 모으는 즐거움
2. **재화 소모 수단** — 캐릭터 유지/성장에 자원을 소비하게 하여 경제 순환 유도
3. **모의투자의 변수** — 예측 불가능한 이벤트를 만들어 투자에 긴장감과 재미를 부여

#### 핵심 설계 원칙

- 캐릭터는 **양날의 검**: 돌발 행동으로 투자를 방해하지만, 알바/제작/스킬로 이득도 줌
- 캐릭터가 **없으면 불리**: 시드머니 수급 수단이 없고, 투자 보조 스킬도 없음
- 캐릭터가 **많을수록 리스크와 리턴 동시 증가**: 생산력 ↑ + 돌발 행동 리스크 ↑
- **문제를 일으키는 존재가 해결책도 만든다**: 돌발 행동을 방어할 소모성 아이템도 캐릭터가 제작

#### 슬롯 규칙 [확정]

- 시설(집) 업그레이드로 슬롯 수 증가 (**다운그레이드 불가** — 비가역적 전략적 선택)
- 초기 수용 인원: **1마리**
- 열린 슬롯에는 **반드시 캐릭터 배치** (직접적 패널티가 아닌 **기회비용** 방식으로 강제)
  - 빈 슬롯 = 알바 수입 0, 아이템 생산 0, 투자 보조 스킬 0
  - 배치된 캐릭터 = 지속적으로 가치를 생산하는 상태
  - → 안 채우면 "돌아가는 생산라인을 비워둔 셈"

#### 캐릭터 활동 시스템 [확정]

플레이어가 **수동으로** 캐릭터별 활동을 배정. 4가지 활동이 존재하며 각각 연계 시설이 있음.

| 활동 | 산출물 | 연계 시설 | 설명 |
|------|--------|-----------|------|
| **아르바이트** | 시드머니 | 외부 (시설 불필요) | 타이쿤 스타일 미니게임. 캐릭터 스텟에 따라 수익 상이 |
| **제작** | 소모성 아이템 | 작업장 (신규 시설) | 손실 케어 쿠폰, 돌발 봉인 부적 등 생산. 스텟/도구에 따라 제작 가능 아이템 상이 |
| **훈련** | 스텟 성장 | 체육관 | 어떤 스텟을 올릴지 선택. 장기적 캐릭터 성장 |
| **휴식** | 컨디션 회복 | 여가시설 | 스트레스/피로도 감소. 컨디션이 낮으면 돌발 행동 확률 ↑ |

**활동 간 균형**: 모든 활동이 의미 있는 선택이 되도록 설계
- 아르바이트만 시키면? → 컨디션 하락 → 돌발 행동 급증
- 훈련만 시키면? → 시드머니 부족 → 투자 자금 없음
- 휴식만 시키면? → 아무것도 생산 안 함 → 기회비용 손실
- 제작만 시키면? → 아이템은 쌓이지만 시드머니 부족

#### 캐릭터 등급 시스템 [확정]

캐릭터는 등급을 가지며, 등급에 따라 **특성 스텟 합계의 범위**가 결정됨.

| 등급 | 스텟 합계 범위 | 생성 비용 | 리롤 비용 | 획득 방법 |
|------|---------------|-----------|-----------|-----------|
| **N (일반)** | 21~28 | 저렴 | 저렴 | 기본 생성, 알바 보상 |
| **R (고급)** | 29~36 | 보통 | 보통 | 이벤트, 상점 |
| **SR (희귀)** | 37~45 | 비쌈 | 비쌈 | 제작, 희귀 드롭 |
| **SSR (영웅)** | 46~55 | 매우 비쌈 | 매우 비쌈 | 희귀 드롭, 거래소 |
| **UR (전설)** | 56~63 | 최고가 | 최고가 | 특수 조건 달성, 거래소 |

- 스텟 합계 범위 내에서 6개 스텟에 **랜덤 분배** (각 스텟 최소 1, 최대 10)
- **리롤**: 등급에 비례하는 비용을 지불하면 합계 범위 내에서 스텟 재분배 가능
- 플레이어가 **커스텀 캐릭터를 출시**할 때도 등급별 생성 비용 + 스텟 랜덤 분배 → 원하는 배분이 나올 때까지 리롤에 재화 소모
- **기본 제공 캐릭터**: 게임에서 미리 디자인한 캐릭터들은 특성 스텟이 사전 지정됨 (캐릭터 개성 표현)

#### 캐릭터 스텟 구조 [확정]

**특성 스텟 7개** (값 범위: 1~10, 생성 시 결정, 정체성 슬롯으로 보정 가능)

**일(Work) 관련 스텟 — 3개**

| 스텟 | 설명 | 주요 영향 |
|------|------|-----------|
| **체력** | 버틸 수 있는 총량 | **컨디션 최대량 결정**. 체력 10이면 컨디션 게이지가 길어서 오래 활동 가능. 체력 1이면 금방 지침 |
| **효율** | 작업 속도와 생산성 | **알바/제작/훈련 전반의 산출량 ↑**. 같은 시간 대비 더 많은 수익, 더 빠른 제작, 더 높은 훈련 효과 |
| **꼼꼼함** | 작업의 정밀도와 완성도 | **제작 품질/성공률 ↑, 알바 실수율 ↓**. 희귀 아이템 제작 확률 ↑, 알바 중 사고 확률 ↓ |

**투자(Investment) 관련 스텟 — 3개**

| 스텟 | 설명 | 주요 영향 |
|------|------|-----------|
| **멘탈** | 정신적 회복력 | **투자 실패 디버프 저항**. 높을수록 연속 손실에도 디버프에 잘 안 걸리고 빠르게 회복 |
| **행동력** | 실행력과 추진력 | **스킬 발동 확률/빈도 ↑**. 높을수록 캐릭터 고유 스킬이 자주 발동 |
| **자제력** | 감정 통제 능력 | **돌발 행동 억제**. 높을수록 돌발 발동 확률 ↓, 강한 등급 비율 ↓. 낮을수록 불안정하지만 변수 많음 |

**공통(Common) 스텟 — 1개**

| 스텟 | 설명 | 주요 영향 |
|------|------|-----------|
| **운** | 보이지 않는 힘 | 일과 투자 **양쪽의 모든 랜덤 요소**에 영향 |

| 영향 범위 | 효과 |
|-----------|------|
| 알바 | 보너스 수익 발생 확률 ↑ |
| 제작 | 대성공(희귀 아이템) 확률 ↑, 제작 실패 확률 ↓ |
| 투자 | 매매 시 유리한 가격 체결 확률 ↑ |
| 돌발 행동 | 돌발 결과가 긍정적일 확률 ↑ (돌발 매수했는데 대박 종목인 경우 등) |
| 드롭 | 아이템/정체성 드롭 확률 ↑ |

**효율 vs 꼼꼼함 차이**
- **효율** = "얼마나 빨리, 많이" → 양적 생산성 (속도, 산출량)
- **꼼꼼함** = "얼마나 잘, 정확하게" → 질적 생산성 (품질, 성공률)
- 효율 ↑ + 꼼꼼함 ↓: 빨리 만들지만 실패도 잦음
- 효율 ↓ + 꼼꼼함 ↑: 느리지만 고품질 아이템 특화

**스텟 간 시너지**

| 조합 | 효과 | 캐릭터 유형 |
|------|------|------------|
| 멘탈 ↑ + 행동력 ↑ | 실패에 강하면서 스킬도 자주 터짐. 안정적 공격형 | "트레이더형" |
| 행동력 ↑ + 자제력 ↓ | 스킬은 자주 터지지만 돌발도 잦음. 대박 or 쪽박 | "겜블러형" |
| 멘탈 ↑ + 자제력 ↑ | 실패에도 흔들리지 않고 돌발도 적음. 안정적 수비형 | "철벽형" |
| 체력 ↑ + 효율 ↑ | 오래 일하면서 산출량도 높음. 양산형 생산 머신 | "일꾼형" |
| 꼼꼼함 ↑ + 운 ↑ | 희귀/고품질 아이템 제작 특화 | "장인형" |
| 효율 ↑ + 꼼꼼함 ↑ | 빠르면서도 정확. 스텟 합계를 많이 먹지만 강력 | "만능형" |
| 자제력 ↓ + 운 ↑ | 돌발이 자주 터지지만 긍정적 결과 확률도 높음 | "트릭스터형" |

**스텟 활용 예시**
- UR "전설의 트레이더": 체력 8, 효율 7, 꼼꼼함 6, 운 9, 멘탈 9, 행동력 10, 자제력 8 (합계 57)
- SSR "일꾼형": 체력 9, 효율 9, 꼼꼼함 7, 운 6, 멘탈 5, 행동력 3, 자제력 7 (합계 46 → 일 특화)
- SR "겜블러형": 체력 4, 효율 3, 꼼꼼함 2, 운 7, 멘탈 6, 행동력 9, 자제력 9 (합계 40 → 스킬 다발동)
- R "장인형": 체력 3, 효율 3, 꼼꼼함 9, 운 7, 멘탈 4, 행동력 2, 자제력 2 (합계 30 → 제작 특화)
- N "신입": 체력 4, 효율 3, 꼼꼼함 3, 운 4, 멘탈 3, 행동력 3, 자제력 5 (합계 25 → 평범)

**컨디션** (실시간 변동값 — 활동/시간에 따라 변화)

| 컨디션 | 범위 | 소모 원인 | 영향 | 회복 수단 |
|--------|------|-----------|------|-----------|
| 피로도 | 0 ~ **체력×10** | 알바, 제작, 훈련 | 낮을수록 모든 활동 효율 ↓, 돌발 확률 ↑ | 휴식(여가시설), 소모성 아이템(진정제) |

- **체력 스텟이 컨디션 최대량을 결정**: 체력 10 → 최대 100, 체력 5 → 최대 50, 체력 1 → 최대 10
- 체력이 낮은 캐릭터는 자주 쉬어야 하고, 체력이 높은 캐릭터는 오래 활동 가능

**컨디션 자동 관리 시스템 [확정]**

플레이어는 캐릭터별로 **휴식 임계값**을 설정할 수 있음 (예: "컨디션 30% 이하면 휴식").

```
컨디션 100% ─────────────────────────── 최상의 컨디션
    │
    │  활동(알바/제작/훈련) → 컨디션 소모
    │
    ▼
 임계값 (플레이어 설정, 캐릭터별 개별 지정 가능)
    │
    │  ⚡ 현재 활동 자동 중단 → 여가시설로 자동 이동 → 휴식 시작
    │
    ▼
 컨디션 회복 중... (여가시설 레벨에 따라 회복 속도 상이)
    │
    │  회복 완료 → 이전 활동으로 자동 복귀
    │
    ▼
컨디션 0% ──────────────────────────── 한계 도달
    │
    │  ⚠️ 돌발 행동 확정 발동 (등급 랜덤, 자제력 낮으면 강한 등급 확률 ↑)
    │  → 강제 휴식 + 일정 시간 활동 불가
    │
    ▼
 활동 불가 상태 해제 → 최소 컨디션으로 복귀
```

**임계값 설정 전략**
- 높게 설정 (예: 60%) → 안전하지만 실제 활동 시간이 줄어 생산성 ↓
- 낮게 설정 (예: 10%) → 생산 시간 극대화, 하지만 컨디션 0 도달 위험 ↑
- 자제력 낮은 캐릭터 → 보수적(높은 임계값) 권장. 컨디션 0 돌발 행동이 강한 등급일 확률 높음
- 자제력 높은 캐릭터 → 공격적(낮은 임계값) 가능. 돌발 행동 발동해도 약한 등급일 확률 높음

**컨디션 소모에 영향을 주는 요소**
- 체력 스텟 → 컨디션 최대량 결정 (오래 일할 수 있음)
- 활동 종류에 따라 소모율 상이 (알바 > 훈련 > 제작)
- 도구 슬롯 아이템에 의한 소모 감소 보정 가능

#### 돌발 행동 시스템 [확정]

- **강도: 중간 (긴장감)** — 포트폴리오 일부에 영향. 한 방에 파산은 없지만 신경 써야 하는 수준
- **발동 조건**: 슬롯에 배치된 캐릭터는 **활동 중이든 휴식 중이든 언제든** 돌발 행동 발동 가능
  - 슬롯에 캐릭터가 있다 = 항상 리스크가 존재한다
- **발동 확률**: 컨디션이 줄어들 때마다 판정. 컨디션이 낮을수록 확률 ↑

```
발동 확률

 높음 │                                    ╱ ← 자제력 낮은 캐릭터
      │                                 ╱
      │                              ╱
      │                     ╱──────╱ ← 일반 캐릭터
      │              ╱─────╱
      │       ╱─────╱
      │╱─────╱──────────────────────── ← 자제력 높은 캐릭터
 낮음 │
      └──────────────────────────────────
      100%                           0%  컨디션
```

- **컨디션 100~70%**: 매우 낮은 확률 (자제력 낮으면 이 구간에서도 간헐적 발동)
- **컨디션 70~30%**: 점진적으로 확률 상승
- **컨디션 30~1%**: 높은 확률 + 강한 등급 돌발 행동 비율 ↑
- **컨디션 0%**: **확정 발동** + 강제 휴식 + 일정 시간 활동 불가

**확률에 영향을 주는 요소**
- 자제력 ↓ → 기본 발동 확률 ↑, 강한 등급 비율 ↑
- 자제력 ↑ → 기본 발동 확률 ↓, 약한 등급 비율 ↑
- 운 ↑ → 돌발 행동 결과가 긍정적일 확률 ↑
- 멘탈 ↓ → 디버프 상태에서 돌발 행동 추가 확률 보정
- 정체성 슬롯 조합에 의한 보정 가능

**방어 수단**
- 컨디션 관리 (임계값 설정으로 자동 휴식)
- 소모성 아이템 (돌발 봉인 부적 — 일정 시간 발동 억제)
- 자제력 높은 캐릭터 배치
- 정체성 슬롯으로 자제력 ↑ 보정

| 돌발 행동 | 강도 | 설명 |
|-----------|------|------|
| 돌발 매수 | 중 | 캐릭터가 랜덤 종목을 소액 매수 |
| 공포 매도 | 중 | 보유 종목 중 하나를 일부 매도 |
| 종목 추천 강요 | 약 | 특정 종목을 사라고 알림 폭탄 (무시 가능하지만 스트레스 ↑) |
| 알바 땡땡이 | 약 | 배정된 알바를 안 하고 수익 0 |
| 제작 실패 | 약 | 제작 중인 아이템이 실패로 소멸 |
| 올인 시도 | 강 | 보유 자산의 일정 %(상한 있음)를 랜덤 종목에 투자 시도 (봉인 아이템으로 방어 가능) |
| ... | ... | (추가 확장 가능) |

#### 멘탈 디버프 시스템 [확정]

투자 손실이 누적되면 캐릭터에게 **부정 상태(디버프)** 가 발생한다. 멘탈 스텟이 이 시스템의 핵심 방어 수단이다.

**발동 조건**

- 투자 손실이 일정 기준 이상 연속/누적될 때 캐릭터별로 디버프 판정
- 멘탈이 높을수록 판정에서 저항 성공 확률 ↑, 디버프 지속 시간 ↓
- 멘탈이 낮을수록 소액 손실에도 쉽게 디버프 발생

**디버프 종류 (예시)**

| 디버프 | 효과 | 회복 조건 |
|--------|------|-----------|
| 의욕 저하 | 활동(알바/제작) 효율 감소 | 시간 경과 또는 휴식 |
| 패닉 | 자제력 임시 하락 → 돌발 행동 확률 증가 | 시간 경과 |
| 번아웃 | 컨디션 회복 속도 감소 | 장시간 휴식 필요 |
| 위축 | 행동력 임시 하락 → 스킬 발동 확률 감소 | 투자 수익 발생 시 해제 |

> 디버프 종류 및 수치는 밸런스 테스트를 통해 조정 예정

**설계 의도**

- 투자 실패가 캐릭터 상태에 직접 영향 → 투자와 캐릭터 육성이 유기적으로 연결
- 멘탈 스텟의 가치 부여: 높은 멘탈 = 연패에도 버티는 캐릭터
- 디버프 → 돌발 행동 확률 증가 → 추가 손실 가능성이라는 악순환 구조로 긴장감 제공
- 정체성 슬롯(멘탈 보정)이나 소모성 아이템으로 대비할 수 있는 전략적 선택지

#### 장비 슬롯 시스템

| 슬롯 | 개수 | 설명 | 예시 |
|------|------|------|------|
| **도구 슬롯** | 1개 | 활동(알바/제작) 능력에 직접 영향 | 고급 칼 (요리 효율 ↑), 고급 키보드 (타자 효율 ↑), 정밀 도구 (제작 품질 ↑) |
| **정체성 슬롯** | 3개 | 특성 스텟에 보정값 부여, 조합에 따라 시너지 발생 | "명상가" (멘탈 +3), "학자" (꼼꼼함 +5), "겜블러" (행동력 +4, 운 +2) |

- 도구와 정체성의 **조합**에 따라 숨겨진 시너지/패시브 효과 발동 가능
- 파밍(알바 보상, 제작, 투자 마일스톤 등)을 통해 도구/정체성 아이템 획득
- **유저 간 도구 및 정체성 거래 가능** (멀티플레이 시)

#### 캐릭터 획득 시스템 (뽑기) [확정]

캐릭터는 **뽑기(가챠)** 를 통해 획득한다. 신규 플레이어에게는 **1회 무료 뽑기권**을 지급하여 바로 게임을 시작할 수 있도록 한다.

**뽑기 종류**

| 구분 | 이름 | 제공 기간 | 최대 등급 | 설명 |
|------|------|-----------|-----------|------|
| 일반 뽑기 | 일반 뽑기 | 상시 | SR | 기본 뽑기. 이후 **특성별**로 세분화하여 복수의 일반 뽑기 배너 운영 예정 (예: 투자 특성 픽업, 일 특성 픽업) |
| 고급 뽑기 | 상시 프리미엄 뽑기 | 상시 | UR | 모든 등급 출현 가능. 프리미엄 뽑기권 소모 |
| 기간제 뽑기 | 기간 한정 프리미엄 뽑기 | 기간 한정 | UR | 모든 등급 출현 가능. 한정 캐릭터/픽업 확률 상승. 주기적으로 새 배너 출시 |

**등급별 출현 확률 (예시)**

| 등급 | 일반 뽑기 | 프리미엄 뽑기 (상시/기간제) |
|------|-----------|---------------------------|
| N | 50% | 30% |
| R | 35% | 35% |
| SR | 15% | 25% |
| SSR | - | 8% |
| UR | - | 2% |

> 확률은 밸런스 테스트를 통해 조정 예정

**뽑기 재화**

- 일반 뽑기권: 알바/제작 보상, 일일 임무, 투자 수익 등으로 획득
- 프리미엄 뽑기권: 마일스톤 달성, 이벤트 보상, 과금(추후 결정) 등으로 획득
- 기간 한정 뽑기도 프리미엄 뽑기권 사용 (별도 재화 없음)

**사용자 제작 캐릭터 vs 뽑기 캐릭터**

- 커스텀 도트로 사용자가 직접 만드는 캐릭터는 **최대 SSR 등급**까지만 가능
- **UR 등급 캐릭터는 뽑기로만 획득 가능** → 뽑기 시스템의 핵심 동기 부여
- 사용자 제작 캐릭터의 등급은 투자한 비용/재료에 따라 결정 (N~SSR 범위)

**향후 확장 계획**

- 일반 뽑기를 특성별로 세분화 (투자 특성 뽑기, 일 특성 뽑기 등)
- 기간 한정 배너 주기적 출시 (시즌/이벤트 연동)
- 천장(Pity) 시스템 도입 검토
- 중복 캐릭터 활용: 유저 제작 캐릭터의 재료로 소모 → 스킬 계승 (핵심 활용처)

#### 캐릭터 성장 시스템 (레벨업) [확정]

캐릭터는 활동(알바, 제작, 훈련 등)을 통해 경험치를 얻고 레벨업한다.

**성장 규칙**

- **최대 레벨 7** — 레벨업할 때마다 **현재 가장 낮은 스텟 +1** (동률 시 처리 규칙은 구현 시 확정)
- 만렙 시 결과적으로 전 스텟 +1 (총 +7)과 동일하지만, 낮은 스텟부터 순차 적용
- 등급/초기 스텟에 관계없이 동일한 보너스
- 개별 스텟 상한은 10 유지 — 이미 10인 스텟은 건너뛰고 다음으로 낮은 스텟에 적용
- 초기 스텟 합이 높은 캐릭터 → 성장 후에도 최종 스펙이 높음 → **좋은 뽑기의 가치 유지**

**등급별 최종 스텟 합 예시 (만렙 보너스 +7 기준)**

| 등급 | 초기 스텟 합 범위 | 만렙 스텟 합 범위 |
|------|-------------------|-------------------|
| N | 15~21 | 22~28 |
| R | 22~32 | 29~39 |
| SR | 33~43 | 40~50 |
| SSR | 44~54 | 51~61 |
| UR | 55~70 | 62~70 (상한 70) |

> 스텟 합 범위는 밸런스 테스트를 통해 조정 예정
> UR 고스텟 캐릭터의 경우 이미 10인 스텟이 있을 수 있어 실제 보너스가 +7 미만이 될 수 있음

**설계 의도**

- 단순 명쾌: 레벨업마다 가장 낮은 스텟 +1, 최대 7레벨
- 약점 보완형 성장: 낮은 스텟부터 올라가므로 자연스러운 밸런스 조정
- 등급 간 격차: 초기 범위 차이가 그대로 유지
- 뽑기 가치: 같은 등급 내에서도 초기 스텟 합이 높은 캐릭터가 최종적으로 유리

#### 캐릭터 처분 시스템 (인수인계) [확정]

캐릭터를 처분하면 **하얗게 불태워지며 파삭 소멸**하는 연출이 재생된다 (번아웃 컨셉).

**인수인계 흐름**

1. 캐릭터 버리기 선택 → 소멸 연출 재생
2. 다른 캐릭터를 지정하여 **"인수인계"** 실행 (시간 소요)
3. 완료 시 **"인수인계 문서"** 아이템 생성 (소모성)
4. 인수인계 문서를 원하는 캐릭터에게 사용 → 경험치 획득

- 인수인계는 훈련/제작과 독립된 **별도 기능**
- 처분 대상의 등급이 높을수록 더 좋은 인수인계 문서 생성
- 인수인계 소요 시간도 처분 대상 등급에 비례

#### 캐릭터 배치 시스템 [확정]

캐릭터는 **메인 그리드 위에 설치한 배치 구역**에 놓아야 활동(알바/제작/훈련 등)과 스킬 효과가 적용된다.

**배치 구역**

- **1칸짜리 배치 구역**을 메인 그리드(시설 그리드) 위 원하는 곳에 설치 (최대 9개)
- 1개의 배치 구역 = 1명의 캐릭터 배치
- 배치 구역은 시설 근처에 분산하거나, 한 곳에 모아서 스킬 시너지를 노릴 수 있음
- 캐릭터의 **시작 위치 = 배치 구역 위치** → 시설까지 최단경로가 개인별로 달라짐

> 최대 배치 구역 수는 **서버 설정으로 조정 가능**

**배치 순서와 스킬 방향**

- 배치 구역 설치 시 **순서 번호(1~9)** 가 매겨지며, 이 순서가 고유 스킬의 전달 방향을 결정
- 스킬 전달은 **배치 순서 기반** (그리드 위치와 무관)
- "다음 N칸" 스킬 → 자기 배치 번호 다음 N명에게 효과 적용
- 마지막 순번 캐릭터의 "다음 N칸" 스킬은 대상이 줄어듦 → 앞쪽 배치가 유리
- 배치 순서 재배열 가능 (캐릭터를 빼지 않고 순서만 변경)
- **같은 뽑기 캐릭터는 1개만 배치 가능** (동일 캐릭터 중복 배치 불가)
- 유저 제작 캐릭터는 각각 별개 캐릭터이므로 같은 스킬이라도 복수 배치 가능

#### 고유 스킬 시스템 [확정]

모든 캐릭터는 **1개의 고유 스킬**을 보유한다. 고유 스킬은 캐릭터 설계 시 결정되며, **행동력** 스텟이 높을수록 발동 확률/빈도가 증가한다.

**스킬 구조 = 타입 + 범위 + 수치**

| 요소 | 설명 |
|------|------|
| **타입** | 스킬의 효과 분류 |
| **범위** | 자신 / 다음 N칸 |
| **수치** | 효과 크기 (캐릭터 설계에 따라 결정) |

**스킬 타입 — 4종**

| 타입 | 설명 | 효과 방향 | 예시 |
|------|------|-----------|------|
| **버프형** | 긍정적 효과 부여 | 자신 또는 다음 N칸에게 이로운 효과 | "집중 근무": 자신 알바 수익 20% ↑ / "멘토링": 다음 2칸 효율 +2 (임시) / "격려": 다음 1칸 컨디션 회복 속도 ↑ |
| **디버프형** | 부정적 효과 부여 | 다음 N칸에게 해로운 효과 | "잔소리": 다음 1칸 컨디션 소모 ↑ / "불안 전파": 다음 2칸 멘탈 임시 하락 / "텐션 브레이커": 다음 1칸 자제력 임시 하락 |
| **양날형** | 버프 + 디버프 동시 | 이득과 손해가 공존 | "과몰입": 자신 효율 +3 대신 컨디션 소모 2배 / "독려": 다음 1칸 생산 ↑ 대신 돌발 확률도 ↑ / "도박꾼": 자신 투자 수익 배율 ↑ 대신 손실 배율도 ↑ |
| **트리거형** | 특정 조건 충족 시 발동 | 조건부 강력 효과 | "역전": 투자 손실 시 다음 투자 수익 배율 ↑ / "위기 대응": 컨디션 30% 이하에서 전 스텟 임시 +2 / "화사한 복귀": 디버프 해제 시 다음 1칸 행동력 임시 ↑ |

> 스킬 타입, 범위 수치, 효과 크기 등은 **설정으로 조정 가능**

**배치 전략 — 타입별 고려**

- **버프형**: 앞쪽 배치 유리 → 뒤쪽 캐릭터들에게 효과 전파
- **디버프형**: 뒤쪽 배치 유리 → "다음 N칸"에 아무도 없으면 피해 없음
- **양날형**: 이득을 감당할 수 있는 캐릭터를 다음 순서에 배치 (예: 멘탈 높은 캐릭터 뒤에)
- **트리거형**: 조건 발동이 잦은 위치에 배치 (예: 컨디션 조건 트리거 → 임계값 낮게 설정한 캐릭터)

```
배치 전략 예시: 1번 → 2번 → 3번 → ... → 9번

1번: 버프형 (다음 3칸) → 2, 3, 4번에게 버프
2번: 양날형 (자신) → 본인 효율 ↑ + 컨디션 소모 ↑
3번: 트리거형 (자신) → 컨디션 30% 이하 시 강력 발동
...
8번: 디버프형 (다음 1칸) → 9번만 영향
9번: 버프형 (자신) → 뒤에 대상 없으니 자기 강화형 배치
```

**설계 의도**

- 바리에이션: 타입 4종 × 범위(자신/다음1~N칸) × 수치 조합으로 다양한 스킬 생성 용이
- 전략적 배치: 버프형은 앞에, 디버프형은 뒤에, 양날형은 조합 고려 → 배치 순서가 핵심 퍼즐
- 양날의 검: 디버프형·양날형 캐릭터는 스텟이 높아도 배치가 어려워 단순 스텟 높은 캐릭터 = 최강이 아님
- 행동력 가치: 좋은 스킬이 있어도 행동력이 낮으면 잘 안 터짐

#### 캐릭터 성향 시스템 (다수결 버프) [확정]

모든 캐릭터는 **투자 성향** 1개를 가진다. 배치된 캐릭터들의 성향 비율에 따라 **다수결 버프**가 전체 배치 캐릭터에게 적용된다.

**성향 종류 (기본 명칭)**

| 성향 | 설명 | 다수일 때 버프 |
|------|------|----------------|
| **한탕충** | "투자해서 벌자, 인생 한방!" | 투자 관련 버프: 투자 수익 배율 ↑, 행동력 보정 ↑, 스킬 발동 시 투자 관련 효과 강화 |
| **월급충** | "일해서 벌자, 꾸준히!" | 일 관련 버프: 알바 수익 ↑, 제작 성공률 ↑, 컨디션 소모 감소 |

> 성향 명칭 및 버프 효과 수치는 **설정으로 변경 가능**

**다수결 규칙**

- 배치된 캐릭터 중 **다수 성향의 버프**가 전원에게 적용
- 예: 배치 9명 중 한탕충 5 / 월급충 4 → 전원 투자 버프
- **동수일 때**: 양쪽 버프를 **모두 적용하되, 각각 절반 효과**
- 예: 배치 8명 중 한탕충 4 / 월급충 4 → 투자 버프 50% + 일 버프 50%

**전략적 요소**

- 캐릭터를 추가/제거하면 다수결이 뒤집힐 수 있음 → 배치 구성 변경 시 항상 성향 비율 확인
- 투자에 집중하고 싶으면 한탕충 비율을 높이고, 생산에 집중하려면 월급충 비율을 높이는 선택
- 동수를 노려서 양쪽 절반 버프를 받는 균형 전략도 가능
- 고유 스킬 배치 순서 + 성향 비율 관리 = 배치 전략의 두 축

### 2.2 커스터마이징 & 거래 시스템

#### 도트 커스터마이징 [확정]

- 비용을 지불하여 **나만의 캐릭터 도트를 직접 찍어 제작** 가능
- 도트 에디터를 통해 외형을 디자인하고 게임 내 캐릭터로 등록
- 제작한 커스텀 캐릭터를 **마켓에 출시하여 판매** 가능

**도트 에디터 — 외부 라이브러리 사용**

- **Piskel** (오픈소스, JS/HTML/CSS 기반) 을 임베드하여 사용
  - GitHub: https://github.com/piskelapp/piskel
  - 임베드: piskel-embed 라이브러리 활용
- 직접 구현하지 않음 (버그 리스크 최소화)
- 게임 내에서 Piskel 에디터를 열어 도트를 편집하고, 완성 시 게임 캐릭터로 등록하는 흐름

**에디터 스펙**

| 항목 | 설정 |
|------|------|
| 캔버스 크기 | **32×32** 픽셀 (고정) |
| 배경 | **투명** (기본값) |
| 애니메이션 프레임 | **4프레임** (idle 등 기본 동작용) |
| 내보내기 | 스프라이트시트 PNG / 애니메이션 GIF |

**필수 기능**

- **팔레트**: 사용한 색상들을 자동 표시, 색상 선택/추가/삭제
- **Undo/Redo**: 단축키 지원 (Ctrl+Z / Ctrl+Y)
- **격자 표시**: 온/오프 토글
- **투명 배경**: 기본값 투명, 체크무늬 배경으로 투명 영역 표시
- **프레임 복사/붙여넣기**: 특정 프레임을 다른 프레임에 복사 (첫 프레임 그린 후 나머지 프레임에 복붙하여 수정하는 워크플로우)
- **실시간 미리보기**: 애니메이션 4프레임 자동 재생 미리보기

> 에디터 캔버스 크기, 프레임 수 등은 **설정으로 조정 가능**

**캐릭터 동작 & 애니메이션 시스템**

캐릭터 동작은 **유저 제작 동작**과 **공통 이펙트**로 나뉜다. 유저는 최소한의 동작만 그리면 되고, 나머지는 시스템이 공통 이펙트로 처리한다. 원하면 공통 이펙트를 커스텀 도트로 교체 가능.

| 동작 | 기본값 | 커스텀 가능 | 설명 |
|------|--------|------------|------|
| **대기 (idle)** | 유저 제작 (필수) | - | 기본 상태. 4프레임 |
| **이동** | 유저 제작 (필수) | - | 보도 따라 걷기. 4프레임 |
| **일 수행** | 공통 이펙트 | O | 알바/제작/훈련 중 표시 |
| **휴식** | 공통 이펙트 | O | 쉬는 모션 |
| **고유 스킬 발동** | 공통 이펙트 | O | 스킬 발동 시 이펙트 |
| **돌발 행동** | 공통 이펙트 | O | 폭주/패닉 등 이펙트 |
| **기쁨** | 공통 이펙트 | O | 투자 수익 등 긍정 리액션 |
| **좌절** | 공통 이펙트 | O | 투자 손실 등 부정 리액션 |
| **소멸** | 공통 이펙트 | X | 처분 시 하얗게 파삭 (시스템 연출) |

- **유저 필수 제작**: idle(4프레임) + 이동(4프레임) = **총 8프레임만 그리면 OK**
- **공통 이펙트**: 캐릭터 스프라이트 위에 이펙트 레이어를 얹는 방식 (느낌표, 별, 불꽃 등)
- **커스텀 교체**: 유저가 원하면 공통 이펙트 대신 직접 그린 도트로 교체 가능
- 소멸 연출은 시스템 공통으로 고정 (번아웃 컨셉 통일)

**사용자 제작 캐릭터 — 제작 흐름** [확정]

```
제작권 구매 → 동일 등급 재료 캐릭터 선택 → 도트 제작 → 스킬 계승 + 스텟 랜덤 → 완성
                                                              ↓
                                                    불만족 시 리롤권으로 스텟 재배정
```

| 단계 | 설명 |
|------|------|
| **1. 제작권 구매** | 등급별 제작권(N/R/SR/SSR)을 게임 재화로 구매. UR 제작권은 없음 (UR은 뽑기 전용) |
| **2. 재료 캐릭터 선택** | 보유 캐릭터 중 **제작권과 동일 등급**의 캐릭터 1개를 재료로 선택. 재료 캐릭터는 소멸 |
| **3. 도트 제작** | Piskel 에디터에서 외형 제작 (idle 4프레임 + 이동 4프레임) |
| **4. 스킬 계승** | 재료 캐릭터의 **고유 스킬과 스킬 타입이 그대로 계승** |
| **5. 스텟 배정** | 해당 등급 범위 내에서 **랜덤 배정** |
| **6. 성향 배정** | 한탕충/월급충 중 **랜덤 배정** |
| **7. 리롤 (선택)** | 스텟이 불만족스러우면 **리롤권을 소모**하여 스텟 재배정 (리롤권은 별도 구매/획득) |

**사용자 제작 캐릭터의 가치**

- 강력한 스킬을 가진 캐릭터를 재료로 쓰면 → 같은 스킬을 가진 새 캐릭터 생성
- 같은 스킬을 여러 캐릭터에 퍼뜨려 운용 가능 → 고등급 뽑기 중복의 핵심 활용처
- 외형은 유저가 직접 디자인 → 개성 + 실용성 둘 다 확보
- 리롤로 원하는 스텟 분포를 노릴 수 있음 → 리롤권 파밍/구매 동기

#### 거래 시스템 (마켓) [확정]

**마켓 등록형** — 판매자가 가격을 정해 등록하면 구매자가 검색/구매하는 방식.

**마켓 분류 — 2종**

**캐릭터 마켓**

| 거래 대상 | 설명 | 표시 정보 |
|-----------|------|-----------|
| **캐릭터** | 키운 캐릭터를 판매 (스텟, 레벨, 장비 포함) | 등급, 스텟 합, 레벨, 성향, 고유 스킬, 장착 장비 |
| **커스텀 도트** | 직접 제작한 캐릭터 외형 판매 | 미리보기 애니메이션, 제작자 이름 |

**아이템 마켓**

| 거래 대상 | 설명 | 표시 정보 |
|-----------|------|-----------|
| **도구 아이템** | 도구 슬롯에 장착하는 아이템 | 효과, 등급 |
| **정체성 아이템** | 정체성 슬롯에 장착하는 아이템 | 스텟 보정값, 등급 |
| **소모성 아이템** | 봉인 부적, 손실 케어 쿠폰 등 | 효과, 수량 |
| **인수인계 문서** | 경험치 재료 | 경험치 양, 원본 캐릭터 등급 |

**마켓 기능**

- **등록**: 판매자가 가격을 설정하여 마켓에 등록. 등록 수수료 발생 (거래 금액의 일정 %)
- **검색/정렬**: 품목 종류, 등급, 가격, 스텟 등으로 필터링/정렬
- **구매**: 구매자가 즉시 구매. 거래 완료 시 판매 금액에서 수수료 차감 후 판매자에게 지급
- **등록 취소**: 판매자가 등록 취소 가능 (등록 수수료는 반환 안 됨)
- **거래 내역**: 구매/판매 이력 조회 가능

**수수료 구조**

| 항목 | 수수료 | 비고 |
|------|--------|------|
| 등록 수수료 | 거래 금액의 일정 % | 등록 시 선불. 취소해도 반환 안 됨 |
| 거래 수수료 | 거래 금액의 일정 % | 거래 완료 시 판매 금액에서 차감 |

> 수수료 비율은 **서버 설정으로 조정 가능**

**싱글/멀티 전환**

- **멀티플레이**: 유저 간 실제 거래. 마켓 본격 활성화
- **싱글(로컬)**: NPC 상점으로 대체 — 고정 가격에 구매/판매 가능

> 마켓 기능, 수수료, NPC 상점 가격 등은 **설정으로 조정 가능**

#### 소모성 아이템 시스템

현실에는 없는 게임만의 특수 아이템들로, 사용 시 소멸되며 전략적 타이밍에 활용하는 재미를 제공.

| 아이템 | 효과 | 카테고리 |
|--------|------|----------|
| **손실 케어 쿠폰** | 매도 시 손실 금액을 일정 % 감면 (예: 10%, 30%, 50%) | 투자 보호 |
| **돌발 행동 봉인 부적** | 일정 시간 동안 캐릭터의 돌발 행동을 억제 | 캐릭터 제어 |
| **스텟 부스터** | 특정 스텟을 일정 시간 동안 일시적으로 상승 | 캐릭터 강화 |
| **알바 더블페이** | 다음 아르바이트 수익 2배 | 수익 강화 |
| **시장 정보지** | 일정 시간 동안 주가 변동 힌트 제공 | 투자 보조 |
| **진정제** | 캐릭터 스트레스 즉시 대폭 감소 | 캐릭터 관리 |
| **행운의 주사위** | 다음 투자 결과에 랜덤 보정 (이득일 수도, 손해일 수도) | 도박/재미 |
| ... | (추가 확장 가능) | ... |

- 아르바이트 보상, 투자 수익 마일스톤, 시설 이벤트 등으로 **획득**
- NPC 상점에서 게임 내 화폐로 **구매** 가능
- **유저 간 거래 가능** (멀티플레이 시)
- 등급별 희귀도 설정으로 파밍 동기 부여 (일반 / 고급 / 희귀 / 전설)

### 2.3 시설 시스템 [확정]

시설을 건설하고 업그레이드하여 캐릭터 활동과 게임 시스템을 해금·강화한다. 시설은 게임의 모든 핵심 시스템과 연결되어 있다.

#### 시설 종류

**1. 집 (캐릭터 배치)**

- 캐릭터별로 **개별 집을 건설**해야 배치 가능 (집 1채 = 캐릭터 1명 배치)
- 최소 1채 ~ 최대 9채 (배치 그리드 최대 캐릭터 수와 연동)
- 집 등급에 따라 **해당 캐릭터에게만** 버프 적용

| 집 등급 | 버프 효과 (예시) |
|---------|-----------------|
| 허름한 집 | 버프 없음. 최소한의 배치만 가능 |
| 일반 집 | 컨디션 회복 속도 소폭 ↑ |
| 좋은 집 | 컨디션 회복 ↑ + 컨디션 최대량 소폭 ↑ |
| 고급 집 | 컨디션 회복 ↑ + 전 스텟 임시 보정 +1 |
| 최고급 집 | 컨디션 회복 대폭 ↑ + 전 스텟 임시 보정 +2 + 돌발 확률 소폭 ↓ |

> 집 등급, 버프 수치는 **설정으로 조정 가능**

**2. 알바 시설 (알바 해금)**

- 특정 알바 시설을 건설해야 **해당 알바를 수행 가능**
- 시설이 없으면 해당 알바 자체가 잠김

| 알바 시설 | 해금되는 알바 | 미니게임 |
|-----------|-------------|----------|
| 주방 | 요리 알바 | 재료 순서 맞추기 (터치/클릭) |
| 주차장 | 발렛파킹 알바 | WASD 차량 조작 주차 |
| 사무실 | 타자 알바 | 키보드 타이핑 |
| 물류센터 | 분류 작업 알바 | A/D(S) 키 분류 |

**3. 알바 효율 설비**

- 알바 수익/속도를 강화하는 설비
- 업그레이드할수록 같은 알바에서 더 많이/빠르게 수익 발생
- 알바 시설별로 개별 효율 설비 존재

**4. 제작 효율 설비**

- 제작 성공률/품질/속도를 강화하는 설비
- 업그레이드할수록 고품질 아이템 제작 확률 ↑, 제작 시간 ↓

**5. 훈련 효율 설비**

- 경험치 획득량/훈련 속도를 강화하는 설비
- 업그레이드할수록 레벨업에 필요한 시간 ↓

**6. 휴식 시설**

- 캐릭터의 컨디션 회복 속도/효율을 강화
- 업그레이드할수록 휴식 시 컨디션 회복량 ↑, 회복 시간 ↓
- 집 등급 버프와 별도로 **전체 캐릭터에게** 적용

**7. 은행**

- 적금 시스템 이용을 위한 시설
- 건설해야 적금 기능 해금
- 업그레이드 시 이용 가능한 적금 상품 종류 ↑, 이자율 보너스 등

**8. 정신병원**

- 멘탈 디버프를 치료/관리하는 시설
- 건설해야 디버프 치료 기능 해금
- 업그레이드 시 치료 속도 ↑, 동시 치료 가능 캐릭터 수 ↑, 치료 비용 ↓

| 정신병원 등급 | 효과 (예시) |
|-------------|------------|
| 기본 | 디버프 1종 치료 가능. 치료 시간 김 |
| 중급 | 디버프 전종 치료 가능. 치료 시간 보통 |
| 고급 | 치료 시간 ↓ + 동시 치료 2명 |
| 최고급 | 치료 시간 대폭 ↓ + 동시 치료 전원 + 디버프 예방 효과 |

> 정신병원 등급, 치료 효과 수치는 **설정으로 조정 가능**

#### 시설 그리드 (부지 배치) [확정]

모든 시설과 캐릭터 배치 구역은 **메인 그리드(부지)** 위에 자유롭게 배치한다.

**메인 그리드 구조**

- 초기 크기는 작게 시작 (예: 8×8)
- **부지 확장 업그레이드**로 그리드 크기를 점진적으로 넓힘
- 그리드 위에 배치 가능한 것: 캐릭터 배치 구역, 시설, 보도

> 메인 그리드 초기 크기, 확장 단계, 확장 비용은 **설정으로 조정 가능**

**시설 크기 (테트리스 배치)**

- 시설마다 **고유 모양(테트리스 블록)** 이 다름 → 그리드 위에 테트리스처럼 끼워맞추기
- 사각형뿐 아니라 **L자, T자, ㄱ자, 일자** 등 다양한 형태
- 좁은 부지에 다양한 모양의 시설을 효율적으로 배치하는 것이 핵심 퍼즐
- 시설 업그레이드 시 모양이 변할 수 있음 → 주변 여유 공간 확보 필요

| 시설 | 칸 수 | 모양 (예시) | 비고 |
|------|-------|------------|------|
| 캐릭터 배치 구역 | 1칸 | ▣ (최대 9개까지 설치) | 1구역 = 1캐릭터. 그리드 어디든 자유 배치 |
| 집 (1채) | 1칸 | ▣ | 캐릭터당 1채씩, 최대 9채 |
| 주방 | 4칸 | ▣▣ (2×2 정사각) | 요리 알바 해금 |
|      |      | ▣▣ |  |
| 주차장 | 4칸 | ▣▣▣ (L자형) | 발렛파킹 알바 해금 |
|        |      | ▣ |  |
| 사무실 | 5칸 | ▣▣▣ (T자형) | 타자 알바 해금 |
|        |      | · ▣ · |  |
|        |      | · ▣ · |  |
| 물류센터 | 3칸 | ▣▣▣ (일자형) | 분류 알바 해금 |
| 알바 효율 설비 | 2칸 | ▣▣ (일자형) | 알바 효율 강화 |
| 제작 효율 설비 | 3칸 | ▣ (ㄱ자형) | 제작 효율 강화 |
|               |      | ▣▣ |  |
| 훈련 효율 설비 | 3칸 | ▣▣▣ (일자형) | 훈련 효율 강화 |
| 휴식 시설 | 4~6칸 | 등급에 따라 모양 변화 | 컨디션 회복 |
| 은행 | 2칸 | ▣▣ (일자형) | 적금 시스템 해금 |
| 정신병원 | 4~6칸 | ▣▣ (등급에 따라 확장) | 디버프 치료 |
|         |       | ▣▣ |  |
| 보도 | 1칸 | · | 경로 연결용 |

> 시설 모양, 칸 수, 업그레이드 시 모양 변화는 **설정으로 조정 가능**
> 시설은 **회전 불가 / 회전 가능** 여부를 설정할 수 있음 (회전 가능 시 배치 자유도 ↑)

**캐릭터 배치 구역**

- **1칸짜리 배치 구역**을 그리드 위 원하는 곳에 설치 (최대 9개)
- 1개의 배치 구역 = 1명의 캐릭터 배치
- 배치 구역은 모아둘 수도, 흩어 놓을 수도 있음 → 자유로운 전략
- 캐릭터의 **시작 위치 = 배치 구역 위치** → 시설까지 이동 거리가 개인별로 다름

**보도 시스템 + 최단경로 이동**

- 캐릭터 배치 구역과 시설 사이를 **보도(1×1 칸)로 연결**해야 캐릭터가 이동 가능
- **보도를 통한 도달 경로가 없으면 해당 시설 이용 불가** (시설이 있어도 못 씀)
- 캐릭터는 현재 위치에서 목표 시설까지 **보도 위의 최단경로(BFS/다익스트라)**를 자동 계산하여 이동
- 이동 시간 = 최단경로 칸 수에 비례 → 짧을수록 효율적
- 보도 등급 업그레이드 시 이동 속도 ↑

**배치 전략**

- **테트리스 퍼즐**: 시설마다 모양이 다르므로(L자, T자, ㄱ자 등) 좁은 부지에 빈 칸 없이 끼워맞추는 것이 중요
- **집중 배치**: 배치 구역 9개를 한 곳에 모음 → 스킬 버프(다음 N칸) 시너지 극대화, 대신 먼 시설까지 이동 시간 ↑
- **분산 배치**: 각 캐릭터를 담당 시설 근처에 배치 → 이동 효율 최고, 대신 스킬 범위가 서로 닿기 어려움
- **클러스터 배치**: 역할별로 2~3명씩 묶어 배치 → 투자팀은 한쪽에 모아 버프 공유, 알바팀은 각 시설 근처에 분산
- 자주 쓰는 시설(알바/휴식)은 배치 구역 근처에, 가끔 쓰는 시설(정신병원/은행)은 멀어도 OK
- 시설 업그레이드 시 **모양 변화**를 대비해 여유 공간 확보 필요
- 보도 경로 설계: 불필요한 우회 없이 주요 시설을 최단으로 연결하는 동선 구성
- 부지가 좁을 때는 배치가 빡빡 → 확장 투자 시기 판단이 중요

```
메인 그리드 배치 예시 (분산 배치 전략)

┌────────────────────────────────────────┐
│ ┌───┐                                  │
│ │집1│ ①·····┌───────┐                  │
│ └───┘       │ 주방  │  ┌───┐           │
│ ┌───┐       │(요리) │  │집3│           │
│ │집2│ ②····└───────┘  └───┘ ③·┌─────┐│
│ └───┘   ·                  ·   │휴식 ││
│         ·    ④ · · · ⑤    ·   │시설 ││
│         ·                  ·   └─────┘│
│   ┌─────┐··⑥···┌────┐  ┌──────┐      │
│   │제작 │      │은행│  │정신  │      │
│   │효율 │   ·  └────┘  │병원  │  ⑨  │
│   └─────┘  ⑦·· ⑧ ···· └──────┘      │
│                                        │
│  ①~⑨ = 캐릭터 배치 구역 (1칸)          │
│  · = 보도                              │
└────────────────────────────────────────┘

배치 전략:
- ①② 주방 근처 → 요리 알바 담당
- ③ 휴식 시설 근처 → 컨디션 낮은 캐릭터
- ④⑤ 중앙 모아 배치 → 스킬 시너지 (다음 N칸)
- ⑥⑦ 제작 효율 근처 → 제작 담당
- ⑧ 정신병원 근처 → 멘탈 약한 캐릭터
- ⑨ 외곽 단독 → 자기 자신형 스킬 캐릭터
```

#### 시설 건설/업그레이드 규칙

- 건설과 업그레이드에는 **게임 재화 + 시간** 소요
- 다운그레이드 불가 (기존 확정 규칙 유지)
- 시설별 최대 등급 존재
- 부지 확장도 재화 + 시간 필요 (고비용·고효과)
- 시설 위치 이동 가능 (재배치 비용 발생)

#### 설계 의도

- 타이쿤 요소: 시설 배치 + 동선 설계가 게임의 핵심 전략 축
- 시설 = 시스템 해금: 알바 시설 없으면 알바 불가, 은행 없으면 적금 불가 → 건설 순서가 전략
- 개인 집 = 캐릭터별 투자: 누구에게 좋은 집을 줄지가 자원 배분 전략
- 부지 확장: "시설 업그레이드 vs 땅 넓히기" 자원 배분 고민
- 동선 효율: 보도 설계와 시설 위치에 따라 캐릭터 생산성이 달라짐
- 정신병원: 멘탈 디버프 시스템의 대응 수단. 시설 투자를 아끼면 디버프 관리가 어려워짐

### 2.4 모의투자 시스템 [확정]

**하이브리드 시세 시스템**: 실제 시세 데이터(`yahoo-finance2`)와 관리자가 추가한 **가상 회사**(GBM 기반 시뮬레이션)가 공존하는 모의투자 시스템. 실제 종목은 뉴스/트렌드에 따라 동료들과 예측하는 재미를, 가상 종목은 관리자가 자유롭게 이벤트와 연동하여 게임성을 부여한다. 지정가/손절/익절 등 실전 주문 유형을 지원한다.

#### 시세 데이터 시스템 (하이브리드)

**1. 실제 시세 — `yahoo-finance2`**

| 항목 | 설명 |
|------|------|
| **데이터 소스** | `yahoo-finance2` npm 라이브러리 (서버 사이드에서 호출) |
| **갱신 주기** | 기본 5분 간격, 관리자가 설정 가능 (최소 1분 ~ 최대 30분) |
| **캐싱** | 서버에서 시세를 캐싱하여 모든 플레이어에게 동일한 데이터 제공 |
| **장 운영 시간** | 실제 장 마감 시에도 게임 내 거래는 가능 (마지막 종가 기준 유지, 장 오픈 시 자동 갱신) |
| **데이터 항목** | 현재가, 시가, 고가, 저가, 거래량, 전일 대비 등락률, 52주 최고/최저 |

**2. 가상 회사 — GBM 시뮬레이션**

| 항목 | 설명 |
|------|------|
| **생성 주체** | 관리자가 에디터에서 가상 회사 추가 (이름, 초기가격, 변동성, 카테고리 설정) |
| **가격 엔진** | 기하브라운운동(GBM) + 평균회귀. 관리자가 변동성(σ), 드리프트(μ) 조정 가능 |
| **갱신 주기** | 실제 종목과 동일 주기로 가격 업데이트 |
| **이벤트 연동** | 게임 내 이벤트가 가상 회사 가격에 직접 영향 (예: "곰돌이 산업 호재" → BEAR 가격 ↑) |
| **혼합 표시** | 종목 리스트에서 실제/가상 구분 태그 표시 (🌐실제 / 🎮가상) |

#### 투자 상품

| 상품 유형 | yahoo-finance2 심볼 예시 | 설명 |
|-----------|--------------------------|------|
| **주식 (Stock)** | AAPL, TSLA, 005930.KS | 실제 기업 주식. 국내/해외 모두 지원 (yahoo-finance2) |
| **ETF** | SPY, QQQ, VOO | 지수 추종 펀드. 분산 투자 용도 (yahoo-finance2) |
| **암호화폐 (Crypto)** | BTC-USD, ETH-USD | 비트코인, 이더리움 등. 24시간 거래 가능 (yahoo-finance2) |
| **환율 (Forex)** | USDKRW=X, EURUSD=X | 통화 쌍 거래. 환율 변동에 베팅 (yahoo-finance2) |
| **가상 회사 (Virtual)** | BEAR, TINY, MOON 등 | 관리자가 만든 가상 종목. GBM 기반 시뮬레이션. 이벤트 연동 |

- 관리자가 에디터에서 **실제 종목** 추가 (yahoo-finance2 심볼 입력) 및 **가상 회사** 생성 (이름/초기가격/변동성 설정)
- 종목별 **거래 수수료율** 설정 가능 (상품 유형별 차등 가능)
- 플레이어는 게임 내 **종목 검색/필터**로 원하는 상품을 찾아 거래
- 종목 리스트에서 실제/가상 구분 표시

#### 종목 화면 (트레이딩 뷰)

| 기능 | 설명 |
|------|------|
| **실시간 차트** | 캔들차트 (1분/5분/15분/1시간/일봉) — 갱신 주기에 맞춰 업데이트 |
| **호가창** | 현재가, 매수/매도 예상 체결가 표시 |
| **종목 정보** | 종목명, 심볼, 현재가, 등락률, 52주 범위, 거래량 |
| **관심 종목** | 플레이어별 관심 종목 즐겨찾기 |
| **뉴스/이벤트** | 주기적 이벤트 시스템과 연동하여 시장 상황 표시 |

#### 주문 유형

| 주문 유형 | 설명 |
|-----------|------|
| **시장가 매수** | 현재 시세로 즉시 매수. 다음 시세 갱신 시점의 가격으로 체결 |
| **시장가 매도** | 현재 시세로 즉시 매도. 다음 시세 갱신 시점의 가격으로 체결 |
| **지정가 매수** | 목표 가격 설정 → 시세가 해당 가격 이하로 내려오면 자동 매수 |
| **지정가 매도** | 목표 가격 설정 → 시세가 해당 가격 이상으로 올라가면 자동 매도 |
| **손절 (Stop Loss)** | 보유 종목에 손절 가격 설정 → 시세가 해당 가격 이하로 내려가면 자동 매도하여 손실 제한 |
| **익절 (Take Profit)** | 보유 종목에 익절 가격 설정 → 시세가 해당 가격 이상으로 올라가면 자동 매도하여 수익 확정 |
| **손절 + 익절 동시 설정** | 하나의 보유 종목에 손절과 익절을 동시에 걸어놓기 (OCO — One Cancels Other) |

- 지정가/손절/익절 주문은 **시세 갱신 시점마다 조건 확인** → 조건 충족 시 자동 체결
- 미체결 주문은 유효 기간(설정값) 경과 시 자동 만료
- 주문 내역에서 **체결 대기 / 체결 완료 / 만료 / 취소** 상태 확인 가능

| 항목 | 설명 |
|------|------|
| **보유 종목 목록** | 종목명, 수량, 평균 매수가, 현재가, 평가손익, 수익률 |
| **총 평가액** | 전체 보유 종목의 시가 평가 합계 |
| **실현 손익** | 매도 완료된 거래의 누적 손익 |
| **미실현 손익** | 현재 보유 중인 종목의 평가 손익 |
| **자산 구성 차트** | 현금 / 주식 / ETF / 암호화폐 / 환율 비율 파이차트 |
| **거래 이력** | 전체 매수/매도 이력 (시간, 종목, 수량, 가격, 수수료, 손익) |

포트폴리오는 하나이며, **플레이어와 캐릭터가 공유**한다.

#### 매매 주체 — 이중 구조

**플레이어 수동 매매**

- 종목 선택, 주문 유형 선택, 매수/매도 타이밍, 투자 금액 모두 플레이어가 직접 결정
- 지정가, 손절, 익절 등 모든 주문 유형 사용 가능
- 캐릭터 없이도 투자 자체는 가능 (단, 스킬 버프 없음)

**캐릭터 자동 매매**

- 배치 그리드에 올린 캐릭터는 **일(알바/제작)과 동시에 투자에도 자동 개입**
- 배치 = 일 + 투자 개입이 패키지 → 분리 불가
- 캐릭터 자동 매매는 **시장가 주문만** 사용 (지정가/손절/익절은 플레이어 전용)
- 캐릭터 스텟에 따라 자동 매매 성과가 달라짐:

| 스텟 | 자동 매매 영향 |
|------|----------------|
| 멘탈 | 연속 손실에도 패닉 매도 안 함. 낮으면 손실 시 비합리적 매도 |
| 행동력 | 스킬 발동 빈도 ↑ → 좋은 타이밍 매매 확률 ↑ |
| 자제력 | 높으면 돌발 매매 억제. 낮으면 충동 매수/매도 빈발 |
| 운 | 자동 매매 결과가 긍정적일 확률 ↑ |

#### 투자 흐름

```
시드머니 확보 (알바/제작)
        │
        ▼
종목 검색/선택 (주식/ETF/암호화폐/환율)
        │
        ▼
주문 유형 선택 (시장가/지정가/손절/익절)
        │
        ├─→ 시장가: 다음 갱신 시점에 즉시 체결
        └─→ 지정가/손절/익절: 조건 충족 시 자동 체결
        │
        ▼
수익/손실 반영 → 포트폴리오 업데이트
        ▲
        │
   캐릭터 자동 매매 / 돌발 행동 개입
```

#### 수익-리스크 밸런스 설계

| 구분 | 규모 | 특성 |
|------|------|------|
| **일 수익 (알바/제작)** | 크고 안정적 | 주 수입원. 캐릭터 배치의 핵심 보상 |
| **캐릭터 자동 매매** | 소소한 손익 | 잘 키우면 플러스, 못 키우면 소폭 마이너스 |
| **돌발 행동** | 큰 손실 가능 | 올인 시도, 패닉 매도 등 한 방에 아픈 피해. 단, 전 재산을 날리는 수준은 아님 |
| **플레이어 수동 매매** | 플레이어 실력에 비례 | 캐릭터 스킬 버프로 유리한 조건 확보 가능 |

> 일 수익 > 캐릭터 자동 매매 손실 (평균) 이 되도록 밸런스 설계
> 돌발 행동은 저빈도·고임팩트 — 터지면 일 수익 여러 번 분량을 날릴 수 있음

#### 돌발 행동과 투자 (강화)

기존 돌발 행동 시스템에서 투자 관련 돌발은 **치명적 강도**로 설계:

| 돌발 행동 | 강도 | 효과 |
|-----------|------|------|
| 돌발 매수 | 중 | 랜덤 종목 소액 시장가 매수 (소소한 손실 또는 운 좋으면 이득) |
| 공포 매도 | 중~강 | 보유 종목 중 하나를 대량 시장가 매도 (타이밍에 따라 큰 손실) |
| 올인 시도 | **강** | 보유 자산의 일정 %(상한 있음, 예: 최대 30%)를 랜덤 종목에 시장가 투자 (봉인 아이템으로 방어 가능) |
| 패닉 셀 | **강** | 보유 종목 일부를 일괄 시장가 매도 — 전량은 아님 (멘탈 디버프 상태에서 발동 확률 ↑) |
| 역배팅 | 강 | 플레이어가 매수한 종목을 매도하거나 그 반대로 행동 |
| 지정가 취소 | 중 | 플레이어가 걸어놓은 지정가/손절/익절 주문을 랜덤으로 취소 |

> 치명적 돌발 행동은 **자제력·컨디션 관리·방어 아이템**으로 대비하는 것이 핵심 전략

#### 방어 수단

- 자제력 높은 캐릭터 육성 → 돌발 발동 확률 자체를 낮춤
- 컨디션 임계값 관리 → 컨디션 저하로 인한 돌발 확률 상승 방지
- **돌발 봉인 부적** (소모성 아이템) → 일정 시간 돌발 행동 억제
- 멘탈 높은 캐릭터 → 연속 손실에도 패닉 셀 저항
- 정체성 슬롯 → 자제력/멘탈 보정
- **손절 주문 활용** → 플레이어가 미리 손절을 걸어두면 돌발 행동으로 인한 최대 손실 제한 가능

#### 관리자 에디터 연동 (투자)

관리자 에디터 **밸런스/설정 관리**에 다음 항목 추가:

| 설정 항목 | 설명 |
|-----------|------|
| 시세 갱신 주기 | yahoo-finance2 API 호출 간격 (기본: 5분, 최소 1분 ~ 최대 30분) |
| 실제 종목 관리 | yahoo-finance2 심볼 추가/삭제, 카테고리(주식/ETF/암호화폐/환율) 분류 |
| 가상 회사 관리 | 가상 회사 생성/수정/삭제 (이름, 초기가격, 변동성σ, 드리프트μ, 카테고리). 이벤트 연동 설정 |
| 상품별 거래 수수료 | 주식 0.1%, ETF 0.05%, 암호화폐 0.2% 등 상품 유형별 수수료율 |
| 지정가 주문 유효 기간 | 미체결 지정가 주문 자동 만료 시간 (기본: 24시간) |
| 캐릭터 자동 매매 규모 | 캐릭터 1회 자동 매매 시 최대 사용 가능 금액 비율 |
| 돌발 올인 상한 | 올인 시도 돌발 행동의 최대 자산 비율 (기본: 30%) |

#### 설계 의도

- **사실적 투자 경험**: 실제 시세 데이터 + 다양한 상품 + 실전 주문 유형으로 진짜 모의투자 느낌
- 이중 매매: 플레이어의 전략적 매매 + 캐릭터의 예측 불가 매매가 공존하는 긴장감
- 배치 = 패키지: 일을 시키려면 투자 리스크도 감수 → 캐릭터가 진정한 "양날의 검"
- 일 > 투자 손실: 평상시에는 일 수익이 자동 매매 손실을 커버 → 캐릭터 배치의 기본 보상 보장
- 돌발 = 치명적: 저빈도·고임팩트로 터지면 아프지만, 대비 수단이 존재 → 아이템 제작/관리 동기
- **손절/익절**: 플레이어만의 전략 도구 — 캐릭터는 시장가만 쓰므로, 고급 주문은 플레이어의 실력 영역

#### 은행 시스템 (적금) [확정]

플레이어의 자산을 안전하게 보호하는 **적금 시스템**. 적금에 넣은 돈은 캐릭터가 접근할 수 없다.

**핵심 규칙**

- 적금에 예치한 자금은 **원금 보장** + 이자
- 적금 자금은 **캐릭터 자동 매매·돌발 행동 대상에서 완전 제외**
- 가용 자금만 투자/매매/돌발 행동에 노출됨
- 다 털려도 적금 해지 후 재시작 가능 → **게임 오버 방지 안전장치**

**적금 상품 구조 (기본값)**

| 만기 | 이자율 | 특징 |
|------|--------|------|
| 10분 | 2% | 초단기 보호. 잠깐 자리 비울 때 |
| 30분 | 5% | 단기 보호. 급할 때 활용 |
| 1시간 | 10% | 빠른 회전. 소규모 보호용 |
| 6시간 | 25% | 반나절 단위. 자리 비울 때 활용 |
| 12시간 | 40% | 하루 절반. 수면 시간 활용 |
| 24시간 | 60% | 하루 단위. 장기 보호 + 최대 이자 |

> 이자율은 **게임 경제 기준** (현실보다 높게 설정). 초반 게임 운영 밸런스에 맞춰 조정 가능

> 적금 상품 종류, 만기, 이자율 등은 **서버 설정에서 자유롭게 추가/수정/삭제 가능**

**중도 해지**

- 만기 전 해지 시 **이자 감소** (원금은 보장)
- 해지 시 감소율은 남은 시간 비율에 따라 차등 적용 (예: 남은 시간이 많을수록 이자 손실 ↑)

**전략적 요소**

- 적금에 많이 넣으면: 안전하지만 투자 가용 자금 ↓ → 수익 기회 감소
- 적금에 적게 넣으면: 투자 자금 풍부하지만 돌발 행동에 크게 노출
- 만기가 긴 상품일수록 이자는 높지만 급히 돈이 필요할 때 해지 손실
- 게임 자산의 몇 %를 적금으로 돌릴지가 **리스크 관리 핵심 전략**

### 2.5 아르바이트 미니게임 시스템 [확정]

게임의 **주 수입원**. 캐릭터를 알바 시설에 배치하면 수익이 발생한다.

#### 수행 모드 — 이중 구조

| 모드 | 설명 | 효율 |
|------|------|------|
| **자동 수행** | 캐릭터를 알바에 배치하면 스텟 기반으로 자동 수행 | 기본 효율 (×1) |
| **수동 수행 (미니게임)** | 플레이어가 직접 미니게임을 플레이 | **기본 효율의 10배** (×10) |

> 수동 수행 효율 배수(기본값: 10배)는 **설정으로 조정 가능**

- 자동: 방치 플레이 가능. 캐릭터 스텟에 따라 수익 결정
- 수동: 미니게임 실력 + 캐릭터 스텟 보정. 시간 투자 대비 큰 보상
- 같은 알바라도 캐릭터 스텟에 따라 자동/수동 모두 수익 차이 발생

#### 알바 종류 — 4종 [확정]

알바별로 해당 **알바 시설을 건설**해야 해금된다. 모든 미니게임은 **30초~1분** 내외로 빠르게 반복 가능하도록 설계한다.

**1. 요리 (주방)**

- **조작**: 터치/클릭
- **규칙**: 화면 위에서 주문서가 내려오면 재료를 올바른 순서대로 터치/클릭해서 요리 완성. 오버쿡 스타일
- **난이도 상승**: 시간이 지날수록 주문 속도 ↑, 재료 종류 ↑, 동시 주문 수 ↑
- **관련 스텟**: 효율(처리 속도 보정), 꼼꼼함(정확도 보정 → 콤보 유지 쉬움)
- **한 판**: 30초~1분

**2. 발렛파킹 (주차장)**

- **조작**: WASD 키로 차량 직접 조작
- **규칙**: 차량이 진입하면 WASD로 몰아서 빈 주차 공간에 주차. 주차된 차량은 **일정 시간(랜덤 범위)이 지나면 자동 디스폰** → 빈 자리가 계속 생기며 무한 반복 가능
- **차량 등급**: 차 종류에 따라 충돌 시 감점 금액이 다름

| 차량 등급 | 충돌 감점 | 주차 보상 | 예시 |
|-----------|----------|-----------|------|
| 저렴한 차 | 소 | 소 | 경차, 중고차 |
| 적당한 차 | 중 | 중 | 세단, SUV |
| 비싼 차 | **대** | 대 | 스포츠카, 외제차 |

- **난이도 상승**: 시간이 지날수록 차량 진입 속도 ↑, 비싼 차 비율 ↑, 대형 차량 등장, 주차장 구조 복잡화
- **감점**: 다른 차량 충돌 시 차량 등급에 따라 알바비 차감, 벽 충돌 시 감점, 주차 시간 초과 시 감점
- **관련 스텟**: 효율(차량 조작 속도 보정), 체력(플레이 지속 시간 보정)
- **한 판**: 30초~1분 (시간제)

**3. 타자 알바 (사무실)**

- **조작**: 키보드 타이핑
- **규칙**: 화면에 문서/단어/숫자가 표시되면 빠르게 정확히 타이핑
- **난이도 상승**: 시간이 지날수록 문장 길이 ↑, 특수문자/숫자 혼합 ↑
- **오타 패널티**: 오타 발생 시 감점 + 재입력 필요
- **시간 보너스**: 작성 시간이 짧을수록 보상 ↑, 길어질수록 보상 감소
- **관련 스텟**: 꼼꼼함(오타 허용 보정 → 소소한 오타 자동 보정), 효율(타이핑 속도 보정)
- **한 판**: 30초~1분

**4. 분류 작업 (물류센터)**

- **조작**: A/D 키 (2분류) 또는 A/S/D 키 (3분류)
- **규칙**: 컨베이어 벨트 위로 상자가 흘러오면 색깔/라벨을 보고 타이밍에 맞춰 해당 키를 눌러 좌/우(또는 좌/중앙/우)로 분류
- **난이도 상승**: A/D 2분류에서 시작 → 일정 점수 달성 시 A/S/D 3분류로 전환, 벨트 속도 ↑, 유사한 색깔/라벨 등장
- **감점**: 잘못된 분류 시 감점, 놓치면 감점
- **관련 스텟**: 꼼꼼함(분류 정확도 보정), 효율(판정 타이밍 여유 보정)
- **한 판**: 30초~1분

> 알바 종류, 미니게임 세부 규칙, 난이도 곡선, 관련 스텟은 **설정으로 조정/추가 가능**

#### 스텟의 영향

| 스텟 | 자동 수행 시 | 수동 수행(미니게임) 시 |
|------|-------------|----------------------|
| **체력** | 알바 지속 시간 ↑ (더 오래 일함) | 미니게임 제한시간 ↑ |
| **효율** | 시간당 수익 ↑ (속도/산출량) | 미니게임 내 처리 속도 보정 ↑ |
| **꼼꼼함** | 보너스 수익 확률 ↑ (품질) | 미니게임 정확도 보정 ↑, 콤보 유지 쉬움 |
| **운** | 추가 보상 드롭 확률 ↑ | 미니게임 중 보너스 이벤트 발생 확률 ↑ |

#### 알바 보상

- **기본 보상**: 게임 재화 (시드머니)
- **추가 보상**: 알바별 고유 재료 아이템, 뽑기권 조각, 장비 재료 등
- 캐릭터 등급/스텟이 높을수록 추가 보상 획득 확률 ↑
- 수동 수행 시 미니게임 성적(점수/콤보)에 따라 추가 보상 등급 결정

#### 알바와 다른 시스템 연동

- **알바 시설 건설** → 해당 알바 해금
- **알바 효율 설비 업그레이드** → 자동/수동 모두 기본 수익 ↑
- **배치 구역 ↔ 알바 시설 거리** → 이동 시간 발생. 가까울수록 효율적
- **컨디션 소모** → 알바하면 컨디션 감소. 컨디션 낮으면 돌발 행동 위험 ↑
- **일 수익 > 투자 손실** → 알바는 안정적 주 수입원. 투자 리스크의 안전판

#### 설계 의도

- 이중 모드: 캐주얼 플레이어는 방치(자동), 적극적 플레이어는 미니게임(수동)으로 큰 보상
- 스텟 연동: 같은 미니게임이라도 캐릭터에 따라 체감 난이도/보상 차이 → 캐릭터 육성 동기
- 확장성: 알바 종류·미니게임 추가가 용이한 구조 (시설 + 미니게임 + 스텟 매핑만 정의)

### 2.6 관리자 에디터 시스템 [확정]

서버 관리자가 **코드 수정 없이 웹 UI로** 게임 컨텐츠를 추가/수정/삭제할 수 있는 관리 도구.

| 공지/이벤트 | 서버 공지사항, 이벤트 생성/관리 |

#### 관리자 에디터 설계 원칙

- **웹 기반 UI** — 서버와 동일 Docker 환경에서 관리자 페이지 제공 (예: `http://<서버IP>:3000/admin`)
- **권한 관리** — 관리자 계정으로만 접근 가능. 일반 유저 접근 차단
- **실시간 반영** — 설정 변경 시 서버 재시작 없이 즉시 적용 (가능한 범위 내)
- **변경 이력** — 누가 언제 무엇을 변경했는지 로그 기록
- **데이터 검증** — 잘못된 값 입력 시 경고 (예: 스텟 합 범위가 겹치는 등급, 확률 합계 100% 초과 등)

### 2.7 유저 간 대출 시스템 [확정]

플레이어끼리 직접 돈을 빌려주고 빌리는 **P2P 대출 시스템**. 차용자(빌리는 사람)가 조건을 제시하고 대출자(빌려주는 사람)가 수락/거절하는 방식. 기한 내 갚지 못하면 강제 회수 및 자산 압류까지 진행되는 하드코어 경제 시스템.

#### 대출 유형

| 유형 | 설명 | 리스크 |
|------|------|--------|
| **담보 대출** | 차용자가 본인 시설/아이템/캐릭터를 담보로 걸고 요청. 담보가 있으므로 저금리 유도 | 미상환 시 담보 자산 압류 — 대출자가 50% 가격에 매각하거나 캐릭터/아이템을 뺏어올 수 있음 |
| **신용 대출** | 담보 없이 요청. 대출자 입장에서 리스크가 크므로 고금리 유도 | 미상환 시 통장 강제 차감 + 자산 압류 |

#### 대출 요청 (차용자 = 돈을 빌리는 플레이어)

차용자가 대출 게시판에 요청 글을 등록한다.

| 설정 항목 | 설명 |
|-----------|------|
| **희망 금액** | 빌리고 싶은 금액 설정 |
| **제시 이자율** | 차용자가 제안하는 이자율 (% 단위) |
| **상환 기한** | 차용자가 제안하는 상환 기한 (최소 10분 ~ 최대 24시간, 적금과 동일한 시간 스케일) |
| **대출 유형** | 담보 대출 / 신용 대출 선택 |
| **담보 자산** | 담보 대출 시: 본인 시설/아이템/캐릭터 중 담보로 걸 자산 선택 (등록 즉시 담보 상태로 잠금) |

#### 대출 수락 (대출자 = 돈을 빌려주는 플레이어)

대출자가 대출 게시판에서 요청을 확인하고 수락 또는 거절한다.

| 단계 | 설명 |
|------|------|
| 1. 요청 목록 조회 | 등록된 대출 요청 목록에서 조건 확인 (금액, 이자율, 기한, 담보 유무, 차용자 채무불이행 이력) |
| 2. 수락 / 거절 | 대출자가 조건을 보고 판단하여 **수락** 또는 **거절** |
| 3. 대출 성립 | 수락 시 → 대출자 계좌에서 차감, 즉시 차용자 계좌에 입금. 상환 타이머 시작 |
| 4. 상환 | 기한 내 차용자가 원금 + 이자를 대출자에게 상환 |

- 요청이 일정 시간(설정값) 동안 수락되지 않으면 자동 만료, 담보 잠금 해제
- 차용자는 수락 전까지 요청 취소 가능 (담보 잠금 해제)

#### 담보 자산 잠금

담보 유형별로 잠금 규칙이 다르다:

| 담보 유형 | 잠금 규칙 | 허용 | 불가 |
|-----------|----------|------|------|
| **시설** | 기능 사용 가능, 업그레이드 가능 | 운영, 업그레이드 | 매각, 이동, 판매 |
| **아이템** | 사용 불가로 묶임 | 인벤토리에서 확인 | 사용, 장착, 판매, 폐기 |
| **캐릭터** | 활동은 가능하지만 장착 변경 불가 | 활동(알바/제작/훈련), 현재 장착 유지 | 장착 변경, 판매, 해고 |

- 담보 자산은 UI에 **자물쇠 아이콘** 표시
- 대출 상환 완료 시 자동으로 잠금 해제
- **미상환 시 대출자(빌려준 사람)의 선택지**:
  - 시설: **50% 가격에 강제 매각** (매각 대금은 대출자에게)
  - 캐릭터/아이템: **대출자가 직접 뺏어올 수 있음** (소유권 이전)

#### 상환 및 불이행 처리

| 단계 | 조건 | 처리 |
|------|------|------|
| **정상 상환** | 기한 내 원금 + 이자 전액 상환 | 대출 종료, 담보 해제 |
| **기한 만료** | 상환 기한 초과 | 즉시 **채무불이행** 상태 진입 |
| **강제 회수 (통장)** | 채무불이행 시 | 차용자 통장(보유 현금)에서 원금 + 이자 + 연체 이자 강제 차감 |
| **적금 해지** | 통장 잔액 부족 시 | 차용자의 적금을 강제 만기 전 해지하여 회수 (이자 감소 적용) |
| **담보 압류** | 담보 대출 + 현금 부족 | 담보 시설물 강제 압류 → 시스템이 자동 매각 → 매각 대금으로 채무 충당 |
| **시설 압류** | 신용 대출 + 현금 부족 | 차용자의 시설물을 가치 순으로 압류 → 자동 매각 → 채무 충당 |
| **잔여 채무** | 모든 자산 압류 후에도 부족 | 남은 채무는 유지, 차용자가 돈을 벌 때마다 일정 비율 자동 상환 (급여 압류) |

#### 연체 이자

- 상환 기한 초과 시 **원래 이자율 × 1.5배**의 연체 이자 발생
- 연체 이자는 시간 경과에 비례하여 누적
- 대출자는 언제든 **강제 회수 버튼**을 눌러 즉시 회수 절차 개시 가능

#### 급여 압류 (잔여 채무 상태)

- 모든 자산이 압류되어도 채무가 남는 경우 발동
- 차용자의 **알바 수입, 투자 수익, 적금 만기금**에서 일정 비율(예: 50%) 자동 차감
- 차용자의 생존을 위해 **최소 보장 금액** 존재 (설정값)
- 잔여 채무 완납 시 급여 압류 해제

#### 대출 제한 및 안전장치

| 항목 | 설명 |
|------|------|
| **동시 대출 제한** | 차용자당 최대 활성 대출 수 제한 (설정값, 기본 3건) |
| **대출 한도** | 차용자의 총 자산 대비 최대 대출 가능 비율 (설정값) |
| **자기 대출 불가** | 본인에게 대출 등록/신청 불가 |
| **대출 이력 공개** | 마켓에서 차용자의 채무불이행 횟수 확인 가능 (신뢰도 지표) |
| **담보 가치 평가** | 담보 시설의 시스템 매각 가치가 대출 금액 이상이어야 담보 인정 |

#### 대출 관련 UI

- **대출 게시판** — 차용자가 등록한 대출 요청 목록 (금액, 이자율, 기한, 유형, 채무불이행 이력 필터)
- **내 대출 현황** — 빌려준/빌린 대출 목록, 상환 진행률, 남은 기한 타이머
- **강제 회수 버튼** — 대출자가 기한 만료 후 즉시 회수 절차 개시
- **채무불이행 알림** — 차용자에게 압류 진행 단계별 알림
- **담보 잠금 표시** — 담보 설정된 시설에 자물쇠 아이콘 + 대출 정보 툴팁

#### 관리자 에디터 연동

관리자 에디터 **밸런스/설정 관리**에 다음 항목 추가:

| 설정 항목 | 설명 |
|-----------|------|
| 최소/최대 상환 기한 | 대출 상환 기한 범위 (기본: 10분~24시간) |
| 동시 대출 제한 | 차용자당 최대 활성 대출 수 (기본: 3건) |
| 대출 한도 비율 | 총 자산 대비 최대 대출 가능 비율 |
| 연체 이자 배율 | 기본 이자율 대비 연체 이자 배수 (기본: 1.5배) |
| 급여 압류 비율 | 잔여 채무 시 수입에서 자동 차감 비율 (기본: 50%) |
| 최소 보장 금액 | 급여 압류 시 차용자에게 남겨주는 최소 금액 |
| 요청 만료 시간 | 대출 요청이 수락되지 않으면 자동 만료되는 시간 (기본: 1시간) |

### 2.8 칭호 시스템 [확정]

시스템이 자동으로 측정 가능한 **명확한 수치 조건**만으로 부여되는 칭호 시스템. 칭호는 플레이어 닉네임 옆에 표시되며, 다른 유저가 확인 가능. 별도의 조건 설계 없이 DB 수치 기반으로만 판정.

#### 칭호 목록

모든 칭호는 **단순 수치 조건**으로만 부여/해제. 조건이 애매하거나 판정이 복잡한 칭호는 만들지 않는다.

| 칭호 | 조건 | 유형 |
|------|------|------|
| **거지** | 순자산 ≤ 0 | 실시간 (조건 벗어나면 해제) |
| **서민** | 순자산 1 ~ N | 실시간 |
| **부자** | 순자산 ≥ N | 실시간 |
| **재벌** | 순자산 ≥ N (상위 구간) | 실시간 |
| **신용불량자** | 채무불이행 상태 (활성 연체 대출 ≥ 1) | 실시간 |
| **사채업자** | 대출 제공 누적 ≥ N건 | 영구 |
| **파산왕** | 순자산 ≤ 0 도달 횟수 ≥ N회 | 영구 |
| **수집가** | 보유 캐릭터 수 ≥ N | 실시간 |
| **UR 헌터** | UR 등급 캐릭터 보유 수 ≥ N | 실시간 |
| **알바왕** | 누적 알바 수입 ≥ N | 영구 |
| **시설왕** | 보유 시설 총 가치 ≥ N | 실시간 |
| **거래왕** | 마켓 거래 누적 ≥ N건 | 영구 |

- N값은 모두 **관리자 에디터에서 설정 가능**
- 실시간 칭호: 현재 상태가 조건을 충족하는 동안만 유지
- 영구 칭호: 한 번 달성하면 영구 보유
- 플레이어는 보유 칭호 중 **1개를 대표 칭호로 선택**하여 닉네임 옆에 표시

#### 관리자 에디터 연동

관리자 에디터 **밸런스/설정 관리**에 다음 항목 추가:

| 설정 항목 | 설명 |
|-----------|------|
| 칭호별 수치 기준 | 각 칭호의 N값 설정 (재산 구간, 누적 횟수 등) |
| 칭호 활성화/비활성화 | 특정 칭호를 사용하지 않도록 끄기 |

### 2.9 실시간 재산 확인 & 랭킹 시스템 [확정]

접속 중인 플레이어의 재산 현황을 실시간으로 확인하고, 서버 내 순위를 경쟁하는 시스템.

#### 실시간 재산 확인

- 접속 중인 플레이어 목록에서 **닉네임, 대표 칭호, 총 자산** 확인 가능
- 총 자산 = 보유 현금 + 적금 원금 + 투자 포트폴리오 평가액 + 시설 매각 가치
- 실시간 갱신 (Socket.io 기반, 일정 간격으로 업데이트)
- 대출 잔액은 총 자산에서 차감하여 **순자산** 기준으로 표시

#### 유저 프로필 (포트폴리오 & 성과 공개)

접속자 목록에서 플레이어를 클릭하면 해당 유저의 **상세 프로필**을 모든 유저가 열람할 수 있다.

| 공개 항목 | 내용 |
|-----------|------|
| **자산 구성** | 현금 / 적금 / 투자 / 시설 비율 파이차트 |
| **포트폴리오** | 보유 종목 목록, 종목별 수량, 평가손익, 수익률 (전체 공개) |
| **투자 성과** | 누적 실현 손익, 누적 수익률, 최대 수익 거래, 최대 손실 거래 |
| **거래 이력** | 최근 N건의 매수/매도 이력 (종목, 수량, 가격, 손익) |
| **보유 캐릭터** | 배치 중인 캐릭터 목록 (이름, 등급, 레벨, 대표 스킬) |
| **시설 현황** | 보유 시설 목록 및 그리드 배치 상태 |
| **칭호 목록** | 보유 칭호 전체 + 대표 칭호 |
| **업적 진행률** | 달성한 업적 목록 |
| **대출 상태** | 활성 대출 건수, 채무불이행 이력 횟수 |

#### 랭킹 보드

| 랭킹 종류 | 기준 | 갱신 주기 |
|-----------|------|-----------|
| **총 자산 랭킹** | 순자산 기준 | 실시간 |
| **투자 수익률 랭킹** | 누적 투자 수익률 | 실시간 |
| **알바왕 랭킹** | 누적 알바 수입 | 실시간 |
| **수집가 랭킹** | 보유 캐릭터 수 / UR 보유 수 | 실시간 |
| **사채업자 랭킹** | 대출 제공 횟수 / 이자 수익 | 실시간 |
| **시설왕 랭킹** | 시설 총 가치 / 그리드 확장 수준 | 실시간 |

#### 랭킹 보상

- 일정 주기(설정값)마다 랭킹 스냅샷을 찍고, 상위 N명에게 보상 지급
- 보상 종류: 재화, 뽑기권, 제작권, 리롤권, 한정 칭호 등 (관리자 설정)

#### 관리자 에디터 연동

관리자 에디터 **밸런스/설정 관리**에 다음 항목 추가:

| 설정 항목 | 설명 |
|-----------|------|
| 랭킹 종류 활성화 | 표시할 랭킹 카테고리 선택 |
| 랭킹 보상 주기 | 보상 지급 주기 (기본: 24시간) |
| 랭킹 보상 내용 | 순위별 보상 아이템/재화 설정 |
| 자산 공개 범위 | 다른 유저에게 공개할 자산 정보 범위 설정 |

### 2.10 주기적 이벤트 시스템 [확정]

일정 주기(기본 30분, 설정 가능)마다 서버 전체에 영향을 미치는 **돌발 이벤트**가 자동 발생하는 시스템. 모든 플레이어에게 동시에 적용되어 긴장감과 공동 경험을 만든다.

#### 이벤트 발생 규칙

| 항목 | 설명 |
|------|------|
| **발생 주기** | 기본 30분마다 1회 (관리자 설정 가능) |
| **발생 방식** | 등록된 이벤트 풀에서 가중치 랜덤 선택 |
| **지속 시간** | 이벤트별 상이 (즉시 적용 ~ 다음 이벤트까지 지속) |
| **사전 알림** | 이벤트 발생 1분 전 전체 알림 (준비 시간) |
| **동시 발생** | 최대 1개 지속 이벤트만 활성 (새 이벤트 발생 시 기존 이벤트 종료) |

#### 이벤트 카테고리

| 카테고리 | 설명 | 예시 |
|----------|------|------|
| **투자 이벤트** | 거래 조건에 영향 (시세 자체는 실제 데이터이므로 조작하지 않음) | 수수료 할인 (거래 수수료 50% 감소), 수수료 폭탄 (수수료 ×2), 거래소 점검 (신규 매수 일시 중단, 매도만 가능), 돌발 행동 폭주 (캐릭터 돌발 매매 확률 ×2) |
| **노동 이벤트** | 알바 보상에 영향 | 야근 수당 (알바 수입 ×2), 파업 (특정 알바 불가), 보너스 시즌 |
| **시설 이벤트** | 시설 운영에 영향 | 정전 (시설 효율 50% 감소), 세일 기간 (시설 건설 비용 할인) |
| **캐릭터 이벤트** | 캐릭터 상태에 영향 | 컨디션 회복 이벤트, 돌발 행동 확률 증가, 스킬 쿨타임 초기화 |
| **경제 이벤트** | 게임 내 재화/시스템에 영향 | 금리 인상 (적금 이자 ×1.5), 세금 부과 (보유 현금의 N% 징수), 대출 금리 상한제 (대출 이자율 일시 제한) |
| **특수 이벤트** | 희귀 발생, 큰 영향 | 잭팟 타임 (뽑기 확률 UP), 복권 추첨 (전원 참여, 1등 당첨자에게 큰 보상), 마켓 프리데이 (마켓 거래 수수료 0%) |

#### 이벤트 표시

- 화면 상단에 **현재 활성 이벤트** 배너 표시 (이름, 효과, 남은 시간)
- 이벤트 발생 시 **전체 알림 팝업** + 효과음
- 이벤트 히스토리 로그 확인 가능 (최근 N건)

#### 관리자 에디터 연동

관리자 에디터에 **이벤트 관리** 메뉴 추가:

| 기능 | 설명 |
|------|------|
| 이벤트 등록 | 이벤트 추가 (이름, 카테고리, 효과, 지속 시간, 발생 가중치) |
| 이벤트 수정/삭제 | 기존 이벤트 편집 또는 삭제 |
| 발생 주기 설정 | 이벤트 발생 간격 조정 (기본: 30분) |
| 사전 알림 시간 | 이벤트 발생 전 알림 시간 (기본: 1분) |
| 이벤트 강제 발생 | 관리자가 수동으로 특정 이벤트 즉시 발생 |

### 2.11 신규유저 가이드 시스템 [확정]

#### 가이드 페이지 구조

| **모의투자** | 수동/자동 매매, 포트폴리오, 돌발 행동, 멘탈 디버프 |
| **아르바이트** | 미니게임 종류별 조작법, 자동/수동 차이, 보상 구조 |
| **시설 관리** | 시설 종류, 그리드 배치, 보도, 업그레이드 |
| **은행 & 대출** | 적금 가입/해지, 대출 요청/수락, 담보, 불이행 처리 |
| **거래소** | 캐릭터 마켓, 아이템 마켓, 등록/구매 방법 |
| **커스터마이징** | 도트 에디터 사용법, 유저 제작 캐릭터 생성 절차 |
| **칭호 & 랭킹** | 칭호 종류/조건, 랭킹 보드, 보상 |
| **이벤트** | 주기적 이벤트 종류, 효과, 대응 전략 |

#### 가이드 페이지 기능

- **검색** — 키워드로 가이드 내 검색
- **북마크** — 자주 보는 항목 즐겨찾기
- **이미지/GIF** — 각 항목에 스크린샷이나 애니메이션 첨부 가능
- **최초 접속 시 팝업** — 신규 유저에게 "가이드를 먼저 보시겠습니까?" 안내 팝업

|------|------|

### 2.12 자동 패치노트 시스템 [확정]

관리자가 에디터에서 컨텐츠를 수정하면 **변경 이력이 자동으로 패치노트로 생성**되어 게시판에 게시되는 시스템. 관리자가 별도로 패치노트를 작성할 필요 없이, 수정 내역이 그대로 유저에게 공유됨.

#### 자동 생성 흐름

```
관리자 에디터에서 컨텐츠 수정
        │
        ▼
변경 이력 자동 기록 (변경 항목, 이전 값, 이후 값, 시간, 관리자)
        │
        ▼
변경 이력을 카테고리별로 묶어 패치노트 자동 생성
        │
        ▼
게시판에 자동 게시 + 접속 중인 유저에게 알림
```

#### 패치노트 자동 생성 규칙

| 항목 | 설명 |
|------|------|
| **생성 단위** | 변경 건이 누적되다가 일정 시간(설정값, 기본 5분) 동안 추가 변경이 없으면 하나의 패치노트로 묶어 생성 |
| **카테고리 분류** | 변경 대상에 따라 자동 태깅 (캐릭터, 스킬, 아이템, 뽑기, 시설, 알바, 밸런스, 이벤트 등) |
| **변경 유형 표시** | 추가(NEW) / 수정(UPDATE) / 삭제(DELETE) / 밸런스 조정(BALANCE) 자동 구분 |
| **제목 자동 생성** | 날짜 + 주요 변경 카테고리 기반 (예: "2026-04-07 캐릭터/밸런스 패치") |
| **버전 넘버링** | 자동 증가 (v1.0.1 → v1.0.2 ...), 관리자가 메이저/마이너 버전 직접 올릴 수도 있음 |

#### 패치노트 형식 (자동 생성 예시)

```
📋 패치노트 v1.2.3 — 2026-04-07 14:30

[캐릭터]
• NEW: 신규 캐릭터 "불꽃이" 추가 (SR등급)
• UPDATE: "얼음이" 체력 스텟 범위 8~12 → 10~14

[밸런스]
• BALANCE: 알바 수익 기본 배율 1.0 → 1.2
• BALANCE: 멘탈 디버프 지속 시간 300초 → 240초

[시설]
• DELETE: "구형 작업장" 삭제 (기존 보유 유저: 상위 시설로 자동 전환)
```

#### 패치노트 게시판

- 게임 내 **공지/패치노트 게시판**에 자동 게시
- 최신 패치노트는 로그인 시 **팝업으로 표시** (한 번 확인하면 다시 안 뜸)
- 패치노트 목록에서 카테고리별 필터 가능
- 각 패치노트에 **댓글 기능** (유저 피드백)

#### 관리자 에디터 연동

관리자 에디터에 **패치노트 관리** 메뉴 추가:

| 기능 | 설명 |
|------|------|
| 패치노트 목록 | 자동 생성된 패치노트 전체 조회 |
| 패치노트 편집 | 자동 생성된 내용을 게시 전/후 수정 가능 |
| 패치노트 삭제 | 잘못 생성된 패치노트 삭제 |
| 버전 관리 | 메이저/마이너 버전 직접 설정 |
| 묶음 대기 시간 | 변경 묶음 대기 시간 설정 (기본: 5분) |
| 알림 설정 | 패치노트 게시 시 유저 알림 활성화/비활성화 |

### 2.13 인게임 채팅 시스템 [확정]

같은 서버에 접속한 플레이어끼리 실시간으로 소통하는 채팅 시스템. Socket.io 기반.

#### 채팅 채널

| 채널 | 설명 |
|------|------|
| **전체 채팅** | 서버 내 모든 접속자에게 보이는 공개 채팅 |
| **귓속말** | 특정 플레이어에게만 보이는 1:1 메시지 (닉네임 클릭 → 귓속말) |

#### 채팅 기능

| 기능 | 설명 |
|------|------|
| **시스템 메시지** | 이벤트 발생, 랭킹 변동, 대출 불이행 등 주요 활동이 채팅창에 자동 표시 (다른 색상으로 구분) |
| **칭호 표시** | 채팅 메시지 앞에 대표 칭호 배지 표시 |
| **이모티콘** | 도트 스타일 이모티콘 지원 (기본 제공 + 관리자가 추가 가능) |
| **채팅 로그** | 최근 N건 유지, 접속 시 최근 메시지 로드 |
| **뮤트/신고** | 특정 유저 채팅 숨기기 (클라이언트 측), 관리자에게 신고 |

#### 채팅 UI

- 화면 하단 또는 측면에 **접을 수 있는 채팅 패널** (게임 화면을 가리지 않도록)
- 새 메시지 도착 시 **알림 뱃지** (채팅 패널이 닫혀 있을 때)
- 시스템 메시지는 배경색으로 구분 (이벤트=노랑, 거래=초록, 대출=빨강 등)

#### 관리자 에디터 연동

| 설정 항목 | 설명 |
|-----------|------|
| 채팅 로그 보관 수 | 서버에 유지할 최근 메시지 수 (기본: 100건) |
| 이모티콘 관리 | 커스텀 이모티콘 추가/삭제 |
| 채팅 금지어 | 필터링할 단어 목록 관리 |
| 시스템 메시지 설정 | 어떤 활동을 채팅창에 표시할지 선택 |

### 2.14 업적/도전과제 시스템 [확정]

명확한 수치 조건으로 판정 가능한 **일회성 도전과제** 시스템. 칭호(현재 상태 반영)와 달리 한 번 달성하면 영구 기록되며, 달성 시 보상을 지급한다.

#### 업적 목록

모든 업적은 **DB 수치로 즉시 판정 가능**한 것만 포함한다.

| 업적 | 조건 | 보상 (관리자 설정) |
|------|------|-------------------|
| **첫 발걸음** | 첫 캐릭터 획득 | 재화 |
| **첫 투자** | 첫 매수 주문 체결 | 재화 |
| **흑자 전환** | 투자 누적 실현 손익 > 0 달성 | 재화 |
| **연속 흑자** | 연속 N회 매도 수익 | 재화 |
| **만렙 달성** | 캐릭터 1마리 레벨 7 도달 | 뽑기권 |
| **UR 수집가** | UR 등급 캐릭터 N마리 보유 | 제작권 |
| **풀 하우스** | 배치 구역 9칸 모두 캐릭터 배치 | 재화 |
| **시설왕** | 시설 N개 건설 완료 | 재화 |
| **그리드 마스터** | 메인 그리드 최대 크기 확장 | 칭호 해금 |
| **알바 장인** | 각 미니게임 N회 이상 수동 완료 | 재화 |
| **첫 대출** | 대출 1건 제공 또는 요청 | 재화 |
| **성실 상환** | 대출 N건 정상 상환 | 재화 |
| **마켓 데뷔** | 마켓에 첫 등록 | 재화 |
| **거래 달인** | 마켓 거래 N건 완료 | 재화 |
| **도트 아티스트** | 유저 제작 캐릭터 1마리 완성 | 재화 |
| **다각화 투자** | 주식/ETF/암호화폐/환율 각 1종목 이상 보유 | 재화 |
| **적금 만기** | 적금 첫 만기 수령 | 재화 |
| **부활** | 순자산 ≤ 0 이후 순자산 ≥ N 복귀 | 특수 칭호 |

#### 업적 규칙

- 조건 충족 시 **자동 달성** + 보상 즉시 지급 + 팝업 알림
- 한 번 달성하면 **영구 기록** (취소 없음)
- 업적 목록 UI에서 달성/미달성 확인 가능 (진행률 표시)
- 채팅창에 업적 달성 시스템 메시지 표시 ("OOO님이 [첫 투자] 업적을 달성했습니다!")

#### 관리자 에디터 연동

| 기능 | 설명 |
|------|------|
| 업적 등록 | 업적 추가 (이름, 조건 수치, 보상 종류/수량) |
| 업적 수정/삭제 | 기존 업적 편집 또는 삭제 |

```

│   │  ┌───────────┐  ┌───────────────┐   │     │
│   │  │ Game API  │  │ Static Files  │   │     │
│   │  │ (Node.js) │  │ (게임 클라이언트)│   │     │
│   │  └─────┬─────┘  └───────────────┘   │     │
│   │        │                            │     │
│   │  ┌─────┴─────┐  ┌───────────────┐   │     │
│   │  │ WebSocket │  │  SQLite   │   │     │
│   │  │ (Socket.io)│  │  (게임 데이터) │   │     │
│   │  └───────────┘  └───────────────┘   │     │
```

### 5.2 서버 실행 방법

```bash
# 1. 이미지 pull
docker pull tinytraders/server:latest

# 2. 서버 실행 (한 줄로 끝)
docker run -d \
  --name tiny-traders \
  -p 3000:3000 \
  -v tiny-traders-data:/app/data \
  -e SERVER_NAME="우리회사 서버" \
  -e MAX_PLAYERS=20 \
  tinytraders/server:latest

# 3. 동료들에게 접속 URL 공유
# → http://<서버IP>:3000
```

- `docker-compose.yml` 제공으로 더 간편한 설정도 지원
- 환경변수로 서버 이름, 최대 인원, 포트 등 커스터마이징 가능

### 5.4 로컬 ↔ 서버 전환 설계

모듈화 원칙에 따라, 로컬 모드와 서버 모드 전환이 매끄럽도록 설계.

```
┌──────────────────────────────────────────────┐
│              Game Client (브라우저)            │
│                                              │
│   ┌──────────────────────────────────────┐   │
│   │        Data Access Layer (DAL)       │   │
│   │  ┌──────────┐    ┌───────────────┐   │   │
│   │  │ Local    │    │   Server      │   │   │
│   │  │ Adapter  │    │   Adapter     │   │   │
│   │  │(IndexedDB)│    │  (REST/WS)   │   │   │
│   │  └──────────┘    └───────────────┘   │   │
│   └──────────────────────────────────────┘   │
│                                              │
│   설정 하나로 Local ↔ Server 전환             │
└──────────────────────────────────────────────┘
```

- **Local Adapter**: IndexedDB에 저장, 오프라인 플레이 가능
- **Server Adapter**: REST API + WebSocket으로 서버와 통신
- 게임 로직은 어댑터에 무관하게 동일하게 동작

### 5.5 Docker 구성 (예시)

SQLite 사용으로 DB 컨테이너 불필요. **서버 컨테이너 1개**로 완결.

```dockerfile
# Dockerfile
FROM node:20-alpine
WORKDIR /app
COPY package.json pnpm-lock.yaml ./
RUN corepack enable && pnpm install --frozen-lockfile
COPY . .
RUN pnpm build
EXPOSE 3000

```

```bash

  --name tiny-traders \

# 백업 (DB 파일 하나 복사로 끝)

```

## 6. 개발 원칙

1. **관리자 에디터 UX 최우선**: 코드 수정 없이 컨텐츠 업데이트가 가능해야 함
2. **모의투자 사실성**: 실제 시세 데이터 + 실전 주문 유형으로 현실감 있는 투자 경험
3. **캡슐화 & 모듈화**: 각 시스템은 독립적으로 개발/테스트 가능하도록 경계를 명확히 설계
4. **테스트 필수**: Vitest 기반 단위/통합 테스트, 특히 재화 관련 로직(주문 체결, 대출 회수, 이자 계산)은 테스트 커버리지 필수
5. **단계적 개발**: 기능 단위로 순차 구현, pnpm + Vite 기반 빠른 빌드

### Phase 6: 소셜 & 경제 시스템
> 예상 기간: 3~4주

- 인게임 채팅 (전체/귓속말, 이모티콘, 시스템 메시지)
- 마켓 (캐릭터 마켓 + 아이템 마켓, 등록형)
- 대출 시스템 (요청/수락, 담보/신용, 강제 회수/압류/급여 압류)
- 칭호 시스템 (DB 수치 기반 자동 부여/해제)
- 랭킹 보드 + 실시간 재산 확인
- 업적/도전과제 (수치 조건 달성 → 보상)
- 주기적 이벤트 시스템 (30분마다, 6개 카테고리)

### Phase 7: 커스터마이징
> 예상 기간: 2~3주

- 도트 에디터 (Piskel 라이브러리 임베드, 32×32, 4프레임)
- 유저 제작 캐릭터 (제작권 → 재료 캐릭터 → 스킬 계승 → 스텟 랜덤/리롤)
- 커스텀 캐릭터 마켓 등록
- 중복 배치 제한 (같은 가챠 캐릭터 1개만, 유저 제작은 별도 개체)

### Phase 8: 가이드 & 패치노트 & 폴리싱
> 예상 기간: 지속

- 신규유저 가이드 페이지 (시스템별 탭, 검색, 북마크)
- 패치노트 게시판 (자동 생성 연동, 댓글)
- 관리자 에디터 추가 메뉴 (알바, 이벤트, 칭호, 업적, 가이드 관리)
- 밸런스 조정, 버그 픽스, 성능 최적화
- UI/UX 개선 (귀여운 도트 톤 일관성, 사운드 추가)

## 7. 게임 엔진 추천

### 추천: Phaser.js (웹 기반)

| 항목 | 평가 |
|------|------|
| **적합도** | ★★★★★ |
| **이유** | npm 생태계와 완벽 호환, yahoo-finance2 바로 사용 가능 |
| **UI/UX** | HTML/CSS 기반 UI를 자유롭게 구성 가능 |
| **모듈화** | Scene 시스템으로 파트별 독립 개발 용이 |
| **멀티플레이** | Socket.io 등으로 확장 가능 |
| **접근성** | 브라우저만 있으면 실행 → 직장 동료와 공유 용이 |
| **커뮤니티** | 대규모 커뮤니티, 풍부한 문서 |

**왜 Phaser.js인가?**

- 이미 `yahoo-finance2`를 npm으로 사용하려는 계획이 있으므로 **JavaScript/TypeScript 기반** 엔진이 자연스러움
- 브라우저에서 바로 실행되므로 **직장 동료들이 설치 없이 플레이 가능**
- 시뮬레이션/타이쿤 장르에 충분한 성능
- React/Vue 등 프론트엔드 프레임워크와 결합하여 **복잡한 UI 구현 가능**

### 대안

| 엔진 | 장점 | 단점 |
|------|------|------|
| **PixiJS** | 렌더링 성능 우수, 자유도 높음 | 게임 프레임워크가 아니라 직접 구현할 것이 많음 |
| **Godot (웹 빌드)** | 강력한 에디터, 내장 기능 풍부 | npm 라이브러리 연동이 복잡, 웹 빌드 용량 큼 |
| **Unity (WebGL)** | 높은 완성도, 에셋 스토어 | 웹 빌드 무겁고, JS 라이브러리 연동 번거로움 |

#### 모듈 캡슐화 & 테스트 전략

시스템 규모가 커질수록 모듈 간 의존성 관리와 테스트 커버리지가 핵심. 각 모듈은 **독립적으로 테스트 가능**하도록 캡슐화한다.

**모듈 캡슐화 원칙:**

| 원칙 | 설명 |
|------|------|
| **명확한 경계** | 각 모듈(character, investment, loan 등)은 자체 타입, 로직, 훅을 내부에 포함 |
| **공개 인터페이스** | 모듈 외부에서 접근하는 함수/타입은 `index.ts`로 명시적 export |
| **의존성 방향** | core → 개별 모듈 → UI 순서. 모듈 간 직접 의존 금지, 이벤트 버스 또는 core를 통해 통신 |
| **상태 격리** | 각 모듈의 Zustand 스토어는 모듈 내부에 선언, 외부에선 훅으로만 접근 |

#### 프론트엔드 라이브러리 역할 분담

| 라이브러리 | 역할 | 사용처 |
|-----------|------|--------|
| **React + TypeScript** | 전체 앱 프레임워크, 타입 안전성 | 모든 UI 레이어 |
| **Phaser.js** | 게임 렌더링 (캐릭터 애니메이션, 시설 그리드, 미니게임) | React 내부 Canvas 컴포넌트로 임베드 |
| **shadcn/ui** | UI 컴포넌트 (버튼, 모달, 탭, 폼, 토스트 등) | 게임 외 모든 UI — 마켓, 대출, 가이드, 관리자 에디터 등 |
| **Tailwind CSS** | 스타일링 | shadcn/ui 기반 + 커스텀 레이아웃 |
| **Zustand** | 클라이언트 전역 상태 | 유저 설정, 게임 로컬 상태, UI 상태 (모달 열림, 선택된 탭 등) |
| **TanStack Query** | 서버 상태 관리 | API 호출 캐싱, 시세 데이터 polling, 포트폴리오/랭킹 실시간 갱신, 낙관적 업데이트 |
| **nuqs** | URL 쿼리 파라미터 상태 | 마켓 필터, 종목 검색, 랭킹 탭, 관리자 에디터 페이지 상태 — 새로고침/공유 시 상태 유지 |
| **Glide Data Grid** | 고성능 데이터 그리드 | 포트폴리오 보유 종목, 거래 이력, 주문 내역, 관리자 에디터 목록, 랭킹 보드, 대출 게시판 |

#### TanStack Query 활용 상세

| 활용처 | Query 방식 | 설명 |
|--------|-----------|------|
| **시세 데이터** | `refetchInterval` 기반 polling | 서버 갱신 주기(기본 5분)에 맞춰 자동 refetch |
| **포트폴리오** | Socket.io invalidation | 매매 체결 시 서버에서 이벤트 → `queryClient.invalidateQueries` |
| **랭킹 보드** | `refetchInterval` polling | 일정 주기로 갱신 |
| **마켓 목록** | 페이지네이션 + 필터 | `useInfiniteQuery`로 무한 스크롤, nuqs 필터와 연동 |
| **대출 게시판** | 실시간 갱신 | 새 요청 등록 시 Socket.io → invalidation |
| **관리자 에디터** | Mutation + 낙관적 업데이트 | 수정 즉시 UI 반영, 서버 확인 후 확정 |
